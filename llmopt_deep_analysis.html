1 研究动机：自然语言优化问题的泛化难题

从自然语言定义并求解优化问题需专家知识；现有 prompt / learning 方法"优化泛化"有限（准确性 + 问题类型通用性不足）。核心问题：如何提升 LLM 在多样化优化问题上的定义与求解泛化能力？

现有方法局限

prompt 法（如 OptiMUS）与 learning 法在问题类型通用性上不足，跨领域（约 20 领域 7 类问题）求解精度有限。

本文主张

LLMOPT 提出五元素统一建模框架 + 多指令微调 + KTO 对齐 + 自纠正，实现端到端自动定义与求解通用优化问题。

核心创新

用 Sets / Parameters / Variables / Objective / Constraints 五要素把自然语言映射为统一结构，挖掘隐含约束，再经多指令 SFT + KTO 对齐 + 自纠正，在 6 个真实数据集上比 SOTA 平均提升 11.08% 求解精度。

2 框架概览：定义 + 求解 + 自纠正

LLMOPT 三组件：数据（五元素定义 + 专家 / GPT-4 协同标注 + 增强）、学习（多指令 SFT + KTO 对齐）、自动测试（五元素 → 求解代码 → 执行 → 自纠正）。基座 Qwen1.5-14B。

端到端流程

自然语言 → 五元素定义（SFT）→ 求解代码生成（SFT）→ KTO 对齐 → 执行 Pyomo → 日志分析 → 必要时回退定义或代码步自纠正，循环至最优解。

3 五元素统一建模框架

五元素：Sets / Parameters / Variables / Objective / Constraints，把自然语言映射为统一结构，并挖掘隐含约束（如正整数约束），提升求解准确率。

中间结构的作用

五元素作为中间结构显式捕获隐含约束，使生成的求解代码更准确（以牺牲少量 ER 为代价换 SA）。消融显示去掉五元素 SA 明显下降（如 Mamo C 64% vs 68%）。

4 多指令微调与对齐

学习阶段含多指令 SFT 与 KTO 对齐。Df_SFT（问题 → 五元素）与 Ds_SFT（问题 / 五元素 → Pyomo 代码）联合训练，最大化条件概率。

多指令 SFT

两个 SFT 数据集联合训练，分别学习"定义"与"求解代码生成"两个子任务，基座 Qwen1.5-14B。

KTO 模型对齐

用 GPT-4 生成 + 专家标注 desirability 的 DKTO 数据做 Kahneman-Tversky 优化对齐，免奖励模型、数据需求低，抑制幻觉。优于 PPO / DPO。

5 自纠正机制

运行日志触发自纠正

自动测试中以运行日志触发自纠正（上限 12 次）：识别执行错误后回退到定义或代码步修正。表 11 显示其优于 GPT-4o 纠正与 12 次重复采样——识别并纠错比重复执行更关键。

必要性的实证

去掉自纠正 ER 暴跌（ComplexOR 94.7%→18.2%）；去掉 KTO 则 SA 与 AST 均劣；去掉五元素 SA 下降。三者缺一不可。

6 建模与求解的整合

端到端数据流

自然语言 → 五元素定义（SFT）→ 求解代码生成（SFT）→ KTO 对齐 → 执行 Pyomo → 日志分析 → 必要时回退定义或代码步自纠正，循环至最优解。三个组件（定义 / 学习 / 测试）串成完整闭环，无显著 seesaw 效应（10 通用任务均 +0.3%）。

7 实验评估：6 数据集 + 20 领域

设置与基线

6 真实数据集（NL4Opt、Mamo Easy/Complex、IndustryOR、NLP4LP、ComplexOR，约 20 领域 7 类问题）；指标 ER / SA / AST。对比 ORLM（learning 法）与 OptiMUS 等 prompt 法。

8 敏感性分析：组件消融

三组件消融

去掉五元素 SA 降（Mamo C 64% vs 68%）；去掉 KTO SA 与 AST 均劣；去掉自纠正 ER 暴跌（ComplexOR 94.7%→18.2%）。Qwen2-72B 比 14B 更优但成本权衡选 14B；无显著 seesaw 效应（10 通用任务均 +0.3%）。

9 代码使用与实战（含 GitHub 源码、安装命令、使用示例）

官方仓库

代码仓库：https://github.com/caigaojiang/LLMOPT（华东师大 + 蚂蚁 + 南大，ICLR 2025，arXiv:2410.13213）。SFT / KTO 代码分别基于 QwenLM/Qwen 与 huggingface/trl，8×A100-80G 训练。

# 1) 克隆仓库
git clone https://github.com/caigaojiang/LLMOPT.git
cd LLMOPT

# 2) 多指令 SFT（定义 + 求解代码）
python sft.py --model Qwen1.5-14B --data Df_SFT,Ds_SFT \
    --base QwenLM/Qwen

# 3) KTO 对齐（desirability 二值标签）
python kto.py --model sft_ckpt --data DKTO --base trl

# 4) 自动测试 + 自纠正（Pyomo 执行）
python solve.py --problem nl.txt --max_retry 12

复现要点：五元素是提升准确的关键中间结构；KTO 免奖励模型、数据需求低；自纠正上限 12 次，识别并纠错比重复执行更关键。基座 14B 在成本 / 精度间取得平衡。

10 总结与未来展望

LLMOPT 用五元素统一建模 + 多指令 SFT + KTO 对齐 + 自纠正，从自然语言端到端自动定义并求解通用优化问题，在 6 个真实数据集上比 SOTA 平均提升 11.08% 求解精度，开源且训推成本低。

Q1：五元素为何提升准确？
作为中间结构显式捕获隐含约束，使代码更准确（以牺牲少量 ER 为代价换 SA）。
Q2：KTO 比 PPO/DPO 好在哪？
免奖励模型、数据需求低，用 desirability 二值标签直接对齐。
Q3：自纠正必要吗？
必要——表 11 显示其优于 GPT-4o 纠正与 12 次重复采样，识别并纠错比重复执行更关键。

局限与未来

依赖高质量专家标注（费时）；自然语言优化数据稀缺；大规模问题数据常以库 / 文件存而非文本。未来可探索自动标注与更大规模问题。
