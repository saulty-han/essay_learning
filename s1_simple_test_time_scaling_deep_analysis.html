1 研究动机：复现 o1 的测试时扩展

LLM 性能长期靠训练期扩展，o1 验证了测试期扩展（test-time scaling）但方法未公开。复现工作（MCTS、多智能体、RL）均未公开复现出清晰的测试期扩展曲线。核心问题：用最简单方法同时实现测试期扩展与强推理。

o1 的方法未公开

测试期扩展被证明有效，但复现工作（MCTS、多智能体、RL）都未能复现出清晰的扩展曲线，缺乏一个简单可复现的基线。

本文主张

s1 用最简单方法：1000 个精心策划的样本微调 + Budget Forcing 测试时控制，完全开源，首个复现出 o1 式扩展曲线的工作。

核心创新

数据策划（质量 / 难度 / 多样性筛选得 s1K）+ Budget Forcing（强制控制思考 token 数），用极简流程在 Qwen2.5-32B 上激活推理能力，实现随思考 token 增多精度上升的扩展曲线。

2 框架概览：数据策划 + 微调 + 控制

三步：(1) 用 Gemini 2.0 Flash Thinking API 生成 59K 三元组（题 + 推理链 + 解）→ 按质量 / 难度 / 多样性过滤得 s1K（1000 题）；(2) 在 Qwen2.5-32B-Instruct 上 SFT 26 分钟（16×H100）；(3) 推理时用 Budget Forcing 控制思考 token 数。

端到端流程

SFT 激活模型已有推理能力 → 推理时按预算强制控制思考长度，最小控制靠 "Wait" 触发二次检查纠错，实现随思考 token 增多精度上升的扩展曲线。

3 数据策划：1000 个样本

s1K 数据筛选三标准：质量（去 API 错误 / 格式问题）、难度（用 7B/32B 两模型筛掉易题 + 推理链长度）、多样性（按 MSC 分类 50 领域均匀采样）。

为何只训 1000 样本

预训练已含海量推理数据，SFT 仅"激活"能力（类 LIMA 浅对齐假设）。59K 全量仅略优于 1K 但耗 394 vs 7 H100 时，故 1K 样本高效且足够。

4 Budget Forcing 测试时控制

Budget Forcing 用两种控制：最大控制（超思考 token 上限则追加 end-of-thinking 分隔符强制结束）与最小控制（抑制结束符、追加 "Wait" 延长思考促自纠）。

最大控制

超思考 token 上限则追加 end-of-thinking 分隔符强制结束，限制推理长度。

最小控制

抑制结束符、追加 "Wait" 延长思考，促使模型二次检查 / 纠错，实现外推（2x "Wait" 把 AIME24 从 50% 推到 57%）。

顺序扩展优于并行

后续计算可基于中间结果迭代精修，多数投票追不上顺序扩展；"Wait" 外推优于 "Hmm" / "Alternatively" / 无字符串。

5 s1-32B 微调

极简训练

在 Qwen2.5-32B-Instruct 上 SFT 26 分钟（16×H100），仅用 s1K 数据。模型 s1-32B 在 AIME24 56.7、MATH500 93.0、GPQA 59.6，超 o1-preview，接近 Gemini 2.0；s1.1（换 DeepSeek-r1 蒸馏）进一步达 MATH500 95.4。

6 数据与推理控制的整合

端到端数据流

数据策划得 s1K → SFT 激活推理能力 → 推理时按预算强制控制思考长度，最小控制靠 "Wait" 触发二次检查纠错，实现随思考 token 增多精度上升的扩展曲线。BF 控制命中率 100%，Scaling 变体为 15 最优。

8 敏感性分析：数据与缩放

数据消融

仅随机 / 仅多样 / 仅最长链均显著变差（AIME24 约 -30%）；59K 全量仅略优于 1K 但耗 394 vs 7 H100 时。方法对比：BF 控制 100%、Scaling 15 最优；条件控制与拒绝采样差（拒绝采样反扩展）。

9 代码使用与实战（含 GitHub 源码、安装命令、使用示例）

官方仓库

完全开源：https://github.com/simplescaling/s1（Stanford / UW / Allen AI / Contextual AI，arXiv:2501.19393）。含模型权重、数据、代码。

# 1) 克隆仓库
git clone https://github.com/simplescaling/s1.git
cd s1

# 2) 准备 s1K 数据（质量/难度/多样性筛选）
python prepare_data.py --src 59k.jsonl --out s1k.jsonl

# 3) SFT 微调 Qwen2.5-32B（16×H100，约 26 分钟）
python train.py --model Qwen2.5-32B-Instruct --data s1k.jsonl

# 4) 推理时用 Budget Forcing 控制思考长度
python infer.py --model s1-32B --budget_forcing --max_think 4096

复现要点：s1K 的三筛选（质量 / 难度 / 多样性）是关键；SFT 仅激活能力；Budget Forcing 用 "Wait" 触发二次检查实现外推。扩展上限受上下文窗口限制（512 步时 12/30 题超窗掉点）。

10 总结与未来展望

s1 用 1000 个精心策划的样本 + Budget Forcing 实现 o1 级推理性能，是首个复现 o1 式测试时扩展曲线的工作，完全开源、样本高效（SFT 仅 26 分钟），s1-32B 在 AIME24 / MATH500 / GPQA 上超过 o1-preview。

Q1：为何只训 1000 样本就有效？
预训练已含海量推理数据，SFT 仅"激活"能力，类 LIMA 浅对齐假设；59K 全量仅略优但耗时 56×。
Q2：顺序扩展为何优于并行？
后续计算可基于中间结果迭代精修，多数投票追不上；"Wait" 触发二次检查纠错实现外推。
Q3：如何突破扩展上限？
可轮换字符串 / 加频率惩罚 / 结合 RL，或用 REBASE 树搜索等并行方法补上下文窗口限制。

局限与未来

扩展最终会趋于平缓；受上下文窗口限制（512 步时 12/30 题超窗掉点）；频繁抑制结束符易陷入重复循环。未来可结合 RL 与树搜索突破窗口限制。
