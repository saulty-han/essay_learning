1 研究动机：MCTS 依赖大量模拟采样

LLM 战略规划能力受质疑，现有研究引入 MCTS 增强规划，但 MCTS 依赖大量模拟采样：① 问答等任务无客观奖励（缺 ground truth）；② 需 >30 次模拟才统计显著，token / 时间成本过高。同时现有多智能体框架要么开放通信易跑题耗 token，要么预定义流程缺乏复用。

模拟成本瓶颈

MCTS 的 simulation 步骤在无客观奖励时既贵又不可靠，需 30+ 次模拟才统计显著，token 与时间成本过高。

多智能体的两难

开放通信易跑题耗 token；预定义流程缺乏复用与灵活适应性。需要一种能动态伸缩、低成本的框架。

本文主张：LLM 专用 MCTS

MASTER 用 LLM 自我评估替代 MCTS 的 simulation，通过多步设计增强奖励客观性，并按任务复杂度动态招募智能体，用修改版 UCT 指导通信。

2 框架概览：推理树 + 动态招募

根智能体初始化问题 → 生成 Thought/Action/Observation（Solution）→ Validation → Assessment（评分 + 置信度）→ 子智能体扩展 → UCT 选择 → 终端智能体 Evaluation → 失败则 Backpropagation。推理树随任务复杂度动态增减智能体数量。

端到端流程

根智能体生成 Context（Solution+Validation+Assessment），子智能体继承父路径全部 Context 继续求解；按修改版 UCT 选最高者扩展（分支数=Number of Branches，扩展上限=Maximum of Expansion），遇终端且通过评估即输出最终答案，否则回传。

3 LLM 自我评估替代模拟

MASTER 消除 MCTS 的 simulation 步骤，用 LLM 自我评估分配奖励。三个设计增强奖励客观性：Validation、Assessment、Backpropagation。

三大设计

Validation：评估前先让 LLM 单独核验 Solution 中事实，提升奖励客观性。Assessment：让 LLM 同时输出分数与置信度，置信度作为权重参与 UCT。Backpropagation：终端智能体评估失败时，用其奖励回传更新路径上所有智能体 Q 值，纠正初始误分配。

4 动态招募智能体

MASTER 按任务复杂度动态调整智能体数量：简单任务用少量智能体，复杂任务扩展更多子智能体并行求解，使计算资源按需分配。

修改版 UCT

标准 UCT 的 exploration 项被改造：去掉固定探索权重会掉点，去探索项 / 仅初始奖励更差。置信度作为权重参与选择，使奖励更客观、分支选择更准。

5 MCTS 引导的通信

Context 继承式通信

子智能体继承父路径全部 Context（Solution+Validation+Assessment）继续求解，避免开放通信的跑题与 token 浪费。修改版 UCT 引导在哪些分支扩展，使通信有结构、可追溯。

回传机制

终端智能体评估失败时，用其奖励回传更新路径上所有智能体 Q 值，纠正初始误分配，使搜索树能从不成功路径中学习。

6 搜索与多智能体的整合

端到端数据流

根智能体生成 Context → 子智能体继承父 Context 继续求解 → 修改版 UCT 选最高者扩展 → 遇终端且通过评估即输出，否则回传更新 Q。推理树随任务复杂度动态增减智能体，每题仅 10,937 token（约 LATS 的 6%）。

7 实验评估：HotpotQA / WebShop / MBPP

设置与基线

数据集 HotpotQA、WebShop、MBPP（各 100 题，3 次平均）。基线 GPT-4(CoT)、ReAct、Reflexion、LATS、MetaGPT、AgentVerse 及 SOTA（Beam Retrieval / AgentKit / AgentCoder）。基于 GPT-4，温度 0.6 生成、0.0 评估。

主结果：HotpotQA 76.0%（超 Beam Retrieval 73.3%）、WebShop 80.0%（超 AgentKit 70.2%）、MBPP 91.0%（接近 AgentCoder 91.8%）。效率：MASTER 每题仅 10,937 token，约 LATS 的 6%。

8 敏感性分析：UCT 变体与超参

消融与超参

修改 UCT 消融：固定探索权重 → 掉点，去探索项 / 仅初始奖励更差；去 Validation → HotpotQA 0.623、去 Assessment → 0.233。分支数=2 最优；扩展上限过低则性能骤降（HotpotQA 上限=1 时 0%）。MBPP 上 UCT 变体无差异（测试用例结果是客观 Observation，置信度≈1）。

10 总结与未来展望

MASTER 用 LLM 自我评估替代 MCTS 的 simulation，通过 Validation / Assessment / Backpropagation 增强奖励客观性，并按任务复杂度动态招募智能体，用修改版 UCT 指导通信，在 HotpotQA / WebShop / MBPP 上达新 SOTA 且 token 成本仅约 LATS 的 6%。

Q1：为何去掉 MCTS 的 simulation？
无客观奖励且模拟成本高，改用 LLM 自评 + 置信度 + 回传替代，大幅降成本。
Q2：分支数 / 扩展上限如何设？
全数据集默认分支=2；HotpotQA/MBPP 扩展=3，WebShop=8。
Q3：MBPP 上 UCT 变体为何无差异？
测试用例结果是客观 Observation，LLM 置信度≈1，各变体退化为相近。

局限与未来

依赖 GPT-4 级打分能力（小模型难用）；需人工配置 Maximum of Expansion 与分支数。未来可探索自适应超参与更小模型的可用性。
