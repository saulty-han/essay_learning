1研究动机：Web 智能体的上下文困境

基于 LLM 的 Web 信息检索智能体展现出巨大潜力，但在长视界任务（long-horizon tasks）上面临一个根本性的上下文管理权衡——这是当前制约其性能的核心瓶颈。

现有方法的两大困境

困境一：ReAct 式追加导致上下文饱和（Context Saturation）。主流的 ReAct 类智能体（Yao et al., 2023）将完整的推理-动作-观察三元组历史追加到上下文中。这种方式保留了信息的完整性，但原始 Web 数据的噪声会迅速累积——经过几十轮交互后，上下文被大量无关细节填满，导致模型做出次优决策。想象一下：在浏览了 50 个网页后，上下文里塞满了每个页面的完整 HTML 片段，关键信息被淹没在噪声中。

困境二：固定式全量摘要导致信息不可逆丢失（Irreversible Loss）。另一类方法在每个步骤对全部历史做机械式摘要来保持上下文整洁。但每次重新摘要都有风险：某个在第 1 步发现的关键细节可能在第 10 步的摘要中被丢弃，且永远无法恢复。假设每步有 1% 的概率丢失关键信息，100 步后该信息的存活概率仅为 0.99^100 ≈ 36.6%；500 步后更是暴跌至 0.99^500 ≈ 0.66%。

人类认知的启示：人类解决问题时既不会保留所有信息，也不会机械地逐步总结。相反，我们执行的是一种"回顾性整合（retrospective consolidation）"——在关键节点回看过去几步的动作，丢弃无关步骤、提炼中间发现、抽象出关键洞察（Newell & Simon, 1972）。这种主动的、有选择性的记忆管理正是长程推理的关键。

AgentFold 的核心洞察

理想的智能体应该将内部上下文视为一个动态的认知工作空间（dynamic cognitive workspace）——需要主动塑造而非被动填充。AgentFold 在每一步执行一个"折叠（Folding）"操作，以多尺度管理历史轨迹：粒度压缩（Granular Condensation）保留重要的细粒度细节；深层整合（Deep Consolidation）将已完成的多步子任务整块抽象掉。这种灵活性使 AgentFold 在 100 轮交互后仅用约 7k token的上下文，并可扩展到 500 轮。

2框架概览：AgentFold 的折叠范式

AgentFold 将上下文管理从"被动副产品"提升为"显式学习组件"。每步推理产生四元组响应：(Thinking, Folding Directive, Explanation, Action)，其中折叠指令立即更新多尺度状态摘要。

3问题定义：ReAct 饱和与固定摘要的困境

ReAct 的饱和问题

ReAct（Reasoning + Acting）范式下，智能体每步输出一个思维（Thought）、动作（Action）、观察（Observation），并将三者全部追加到上下文中。设每步产生平均 L 个 token，T 步后上下文大小为 O(T·L)。对于 Web 浏览场景，L 通常包含大量原始网页文本，L 值可达数千 token，导致 T=30~50 时即达到上下文窗口上限。

固定摘要的信息丢失

另一种方案是在每步对整个历史做一次摘要。这虽然控制了上下文长度，但存在不可逆的信息损失风险。更严重的是，摘要策略是预先定义的、静态的，无法根据当前任务的特定需求灵活调整——有些步骤的细节可能对未来至关重要，却被过早压缩掉了。

关键权衡公式：若每步摘要有 ε 概率丢失关键信息，则 T 步后存活概率 = (1-ε)T
AgentFold 突破口：不采用"全部保留"或"统一压缩"，而是在每一步自主决定如何处理历史——可以只压缩最新一步（粒度压缩），也可以把已完成的整个子任务整块抽象掉（深层整合）。决策权交给模型本身，通过 SFT 学会这种能力。

4上下文结构：多尺度状态摘要 + 最新交互

Ct = Multi-Scale State Summaries (St) ⊕ Latest Interaction (It)

AgentFold 的上下文由两个互补部分构成：

① 多尺度状态摘要 St（Multi-Scale State Summaries）：一组按时间分块的压缩记录，每个块代表一段已被"折叠"的历史轨迹。例如 "[Folded Step 1~3]: 用户查询了 X 主题，初步发现 A 和 B 两个方向"；"[Folded Step 4~7]: 深入探索了 A 方向，找到候选 C"。这些块是噪声过滤后的、高度浓缩的，为长期规划提供全局视角。

② 最新交互 It（Latest Interaction）：最近一步的完整、未压缩的记录（包含思考过程、工具调用、工具返回结果）。它提供了短期决策所需的最高分辨率细节，确保不会因过度压缩而遗漏即时相关信息。

📌 图 2（对应论文 Figure 2）：中间步骤的上下文结构示意 左侧为输入上下文（问题 + 工具列表 + 多尺度摘要 + 最新交互）；右侧为 AgentFold 四部分响应（Think / Fold / Explain / Tool Call），以及两种折叠模式的对比。

为什么这种设计优于纯追加或纯摘要？

Latest Interaction 解决了短期精确性——智能体在做下一步决策前能看到上一步的所有原始细节（包括工具返回的错误信息、未预期的数据等）。Multi-Scale Summaries 解决了长期连贯性——即使过了 100 步，智能体仍能在压缩的摘要中找到早期的重要发现，而不需要在海量的原始记录中翻找。两者的结合实现了"既见树木又见森林"的效果。

5响应格式：思考 / 折叠指令 / 解释 / 动作

Rt = AgentFold(Ct; θ) → (tht, ft, et, at)

每步 AgentFold 输出一个连贯的四元组文本块，解析为四个组件：

(1) Thinking tht：详细的 Chain-of-Thought 内部独白。智能体在此分析当前上下文 Ct，同时权衡两个决策维度——如何折叠历史上下文以及下一步采取什么动作。其他三个结构化分量均由此推导而出。

(2) Folding Directive ft：智能体对多尺度状态摘要 St−2 的显式修改命令。格式为 JSON：{ "range": [k, t-1], "summary": "σt" }，其中 k 是要折叠的起始步骤 ID，σt 是替换用的摘要文本。支持两种模式：
• ⓵ Granular Condensation（k=t-1）：只折叠最新的单步交互，保留最高分辨率。适用于增量步骤——将一条冗长的原始记录转为紧凑摘要（如"[压缩步骤5] 发现新候选 XYZ 需进一步探索"）。
• ⓶ Deep Consolidation（k<t-1）：折叠多个连续步骤为一个粗粒度摘要。适用于已完成的子任务——当某段探索已得出明确结论（如"方向 A 无效"），可将整段轨迹抽象为一句总结，彻底清理无关细节。

(3) Explanation et：对下一步动作动机的自然语言解释。这不仅增强了可解释性，更重要的是——决定"哪些内容值得保留"的最佳时机恰恰是规划新动作的时刻。这个紧密耦合确保了 AgentFold 的行为既有目的又高效。

(4) Action at：推进任务的下一步动作（如调用搜索工具、访问 URL 等），执行后将工具返回结果追加为新的 Latest Interaction。

关键设计哲学：上下文管理不是推理的"副作用"，而是与动作规划同等重要的核心学习组件。通过将折叠指令纳入响应的四元组，AgentFold 在训练中学会了何时保留细节、何时抽象整段历史——这是一种内化的、自适应的能力，而非依赖外部规则或固定模板。

6训练方法：Fold-Generator 数据流水线 + SFT

6.1 为什么需要专门的训练？

AgentFold 要求模型输出准确、结构化的多元件响应——即使是 GPT-4 级别的模型也无法通过纯提示工程可靠地做到这一点。因此作者构建了一个专门的数据收集管道 Fold-Generator，基于强大的开源 LLM 生成训练轨迹，再通过 SFT 将这种复杂的多步推理蒸馏进目标模型。

6.2 Fold-Generator 数据收集流程

① 问题集：使用 WebSailor (Li et al., 2025a) 相同的问题集以确保公平比较。

② 生成 + 过滤：Fold-Generator 对每个问题生成完整轨迹，然后通过拒绝采样（rejection sampling）严格筛选：
• 不符合要求格式的步骤 → 丢弃
• 包含过多环境错误的轨迹 → 丢弃
确保每个数据点都是清晰正确的推理范例

③ 输出形式：高质量的交互对集合 { (Ct, R*t) }N，其中 Ct 是结构化上下文，R*t 是经验证的金标准响应，N 是所有问题的总交互步数。

④ SFT 训练：使用这些数据进行常规监督微调，目标是将复杂的、多步验证过的推理蒸馏为单一高效的前向传播。

SFT 方法的三大优势

① 能力内化：将"折叠"从脆弱的依赖提示词的外部指令转变为鲁棒的、内在化的模型技能。

② 效率提升：SFT 将计算密集的"生成-过滤"策略蒸馏到最终模型的权重中，使 AgentFold 推理时比用于数据收集的通用 LLM 更高效。

③ 可控透明：整个流水线基于开源模型构建，保持了对数据和训练过程的完全控制，便于详细检查与迭代改进。

上下文的非单调增长特性

在扩展到 500 步上限的实验中，AgentFold 的上下文大多保持在 20k token 以内，且并非单调增长。这是因为当一段长探索被证明无效时，智能体会对该失败子轨迹进行深层整合——将整段无关历史抽象掉并转向新策略，往往使上下文重置为更紧凑的状态。这种自纠正式的上下文管理是 AgentFold 能够持续数百步交互的根本原因。

10总结与未来展望

Q: AgentFold 与其他上下文压缩方法（如 MemGPT、InfLLM）有何不同？
A: MemGPT/InfLLM 采用的是被动触发式的压缩策略——当上下文快满时才做摘要。而 AgentFold 的折叠是主动的、每步执行的，且折叠决策与动作规划紧密耦合。更重要的是，AgentFold 支持多尺度折叠（可压缩单步也可压缩多步子任务），而大多数替代方法只有单一压缩模式。此外，折叠能力是通过 SFT 内化到模型中的，而非依赖外部提示模板。
Q: 30B 模型真的能超越 671B 吗？这合理吗？
A: 这正是本文的核心论点——架构创新比单纯堆砌参数更重要。671B 的 DeepSeek 用 ReAct 范式运行，在 30-50 步后上下文就饱和了，后续步骤质量急剧下降。而 AgentFold 通过折叠保持了上下文的精炼和聚焦，使得 30B 模型能在 100+ 步后仍保持高效推理。类比：一个有条理的人（AgentFold）比一个记忆力强但不会整理笔记的人（大模型 ReAct）更能完成长线任务。
Q: AgentFold 可以应用到 Web 搜索以外的领域吗？
A: 可以。论文的核心贡献是通用的主动上下文管理范式，理论上可用于任何需要长序列推理的任务：代码生成（多文件重构）、数学证明（长链推导）、多轮对话系统、甚至本项目的其他论文（如 D-Bot 的诊断流水线也面临类似的长上下文问题）。作者已在 DeepResearch 产品中将其应用于实际的研究助手场景。

当前局限

• Fold-Generator 数据质量依赖底层 LLM 能力，可能引入偏差；

• 仅在信息检索类基准上验证，尚未在代码生成/数学等领域广泛测试；

• 折叠决策的可解释性有待增强——目前难以直观判断每步为何选择某种折叠模式；

• 仍需外部工具调用（搜索/浏览），工具本身的错误会影响整体表现。

《AgentFold：具有主动上下文管理的长视界 Web 智能体》，作者：叶睿、张忠望、李宽、殷慧峰（*共同第一作者，通讯作者）、陶正伟、赵一达、苏良才、张立文、乔子宇、王新宇、谢鹏军、黄飞、陈思恒、周京仁、蒋勇（*通讯作者），阿里巴巴通义实验室。

摘要

基于 LLM 的 Web 智能体在信息检索方面展现出巨大潜力，然而它们在长视界任务上的有效性受到上下文管理中一个根本性权衡的阻碍。主流的基于 ReAct 的智能体因其累积了嘈杂的原始历史而遭受上下文饱和之苦，而那些在每步固定地概括全部历史的方法则面临着关键细节不可逆丢失的风险。针对这些问题，我们提出 AgentFold——一种以主动上下文管理为中心的新型智能体范式，灵感来源于人类的回顾性整合（retrospective consolidation）认知过程。AgentFold 将其上下文视为一个需主动雕琢的动态认知工作空间，而非一个待填充的被动日志。在每一步，它学习执行一个"折叠（folding）"操作，以多尺度管理其历史轨迹：它可以执行粒度压缩以保存至关重要的细粒度细节，或执行深层整合以将整个已完成的多步子任务抽象掉。在主流基准上的结果是惊人的：仅需简单的监督微调（无需持续预训练或强化学习），我们的 AgentFold-30B-A3B 智能体在 BrowseComp 上达到 36.2%，在 BrowseComp-ZH 上达到 47.3%。值得注意的是，这一表现不仅超越或匹配了规模大得多的开源模型（如 DeepSeek-V3.1-671B-A37B），还超越了领先的专有智能体（如 OpenAI 的 o4-mini）。AgentFold 在复杂任务求解期间主动折叠其工作上下文。

1 引言

有效寻求和综合 Web 信息的能力是现代进步的基础性前提。然而这一关键过程从根本上受限于人类固有的认知容量与耐力极限。基于 LLM 的 Web 智能体标志着范式的转变，提供了一种能够超越这些边界、不知疲倦地在数字景观中导航并显著增强复杂信息搜寻任务效率与效果的系统。然而，当代 Web 智能体的一个关键挑战在于在上下文全面性与简洁性之间取得有效平衡——这一权衡显著影响其性能，尤其在长视界任务上。(1) 主流的基于 ReAct 的智能体将推理-动作-观察三元组的全部历史累积在上下文中，保留了信息完整性但因原始 Web 数据的大量噪声而严重受苦于次优动作。(2) 反之，近期那些在每步机械地概括全部历史的方法维持了整洁的上下文，但在任何单次摘要阶段都面临关键细节的过早且不可逆丢失的风险。这些根本性局限性揭示了当前方法论中的一个关键缺口，表明有必要开发具有高级上下文管理的下一代智能体范式。

在本文中，我们认为理想智能体应像人类的心理草稿纸那样管理其内部上下文：一个需主动管理的工作空间，而非被动填充的空间。人类问题解决既不涉及对所有信息的穷尽式保留，也不涉及刚性的逐步总结。相反，它是一种在关键时刻执行的有纪律的回顾性整合过程。这涉及一种动态的"回看"机制：在若干个动作之后，丢弃无关步骤、提炼中间发现、抽象出关键洞察。这种自我纠错式的整合行为是实现有效且持续的推理的关键能力——我们认为这对智能体有效的长视界推理与探索至关重要。

遵循这一精神，我们介绍 AgentFold——一种被设计为在任务执行期间主动且智能地"折叠"上下文片段的智能体架构。它不是操作单一的日志，而是操作一个动态的、迭代展开的轨迹。在典型步骤中，其推理产生一个包含以下内容的多部分响应：管理历史状态摘要的折叠指令、其思维过程的解释以及下一个动作。折叠指令被立即应用以更新未来的多尺度状态摘要，而解释、执行的动作及其产生的观察形成新的最新交互供下一周期使用。此过程重复直到智能体判定其已收集足够信息以提供准确的最终答案。初始步骤是一个特例，由于缺少先前的历史而省略折叠指令。

此操作循环建立了一个强大的感知→推理→折叠→行动循环，其中上下文整理是一个显式的、学习到的步骤，而非被动的副产品。通过将明确定义的认知工作空间与智能体操纵它的自主性相结合，AgentFold 直接解决了保留细粒度细节与防止上下文膨胀之间的关键权衡，使更聚焦且高效的推理过程成为可能，从而应对复杂的长视界挑战。

4 实验

实现。我们基于开源 LLM Qwen3-30B-A3B-Instruct-2507（总参数 30B，激活 3B）训练 AgentFold。我们将最大工具调用数设为 100，超出将被强制终止。

基准。我们考虑 3 个信息检索基准：BrowseComp（英文）、BrowseComp-ZH（中文）和 WideSearch-en。

主要结果。如表 1 所示，AgentFold-30B-A3B 为开源智能体建立了新的最先进水平，并与领先的专有系统高度竞争。它在开源领域确立了统治地位，击败了规模大达 20 倍的模型——在 BrowseComp 上得分 36.2%（对抗 DeepSeek-V3.1 的 671B 参数量的 30.0%）。此外，AgentFold 在最高水平证明了自身实力：在 WideSearch 上获得最佳总分 62.1%，超越了所有专有智能体包括 OpenAI-o3 和 Claude-4-Sonnet。这些结果凸显了我们架构创新的深远影响，展示了有效的上下文管理如何弥合与大得多的模型之间的性能差距。

上下文动态。我们分析了 BrowseComp 上 200 条轨迹。绘制了每轮存活的轨迹数量 |Tt|（灰柱）与这些相同轨迹的平均上下文 token 数 At（蓝曲线）。结果表明 AgentFold 的上下文在交互过程中保持稳定增长而不爆炸。

交互步数的扩展性。基于其在紧凑上下文方面的发现，我们在 BrowseComp 上测试了将交互步数扩展到 256 的情况（与 355B GLM-4.5 基线对比）。结果展示两大优势：(1) 我们的较小模型在所有可比的步数限制下持续优于 355B 基线；(2) GLM-4.5 智能体的性能在 64 步后饱和并下降（因其追加式上下文已满），而 AgentFold 的准确率稳步提升直至 256 步，展示了良好的扩展属性。

为进一步探测这些极限，我们进行了扩展实验，将最大步数增至 500。结果显示上下文大多保持在 20k token 以下，且非单调增长。这一行为是 AgentFold 识别并从死胡同中恢复能力的直接结果。当一段漫长的探究被证明无效时，智能体能对整个失败的子轨迹进行深层整合——将一段长的、无关的历史抽象掉同时转向新策略，往往将上下文重置为更紧凑的状态。这展示了一种复杂的、自纠正形式的上下文管理。

此实验确认了 AgentFold 的主动上下文管理是解锁长视界任务求解的关键。它展示了智能体参与真正延展交互——可能持续数百步——以执行所需的广泛且深入的 Web 探索的巨大潜力，而这种探索目前远非传统智能体架构所能企及。

结论

我们介绍了 AgentFold——一种新型智能体架构，其核心是通过主动的上下文管理解决长视界任务中的上下文权衡难题。通过将上下文视为需主动管理的动态认知工作空间，AgentFold 实现了在保持关键细节的同时防止上下文膨胀。其简洁的上下文使其能够在数百步的交互中保持高效推理。在主流基准上的结果令人印象深刻：仅需简单 SFT（无持续预训练或 RL），我们的 30B 模型超越了规模大 20 倍以上的模型，并与顶级专有系统相匹敌。这标志着从静态预定义上下文策略的智能体到自我感知的知识管理者的概念飞跃。通过将上下文整理作为可学习的核心动作整合进来，AgentFold 学到了关于记住什么、抽象什么、丢弃什么的复杂任务特定策略。这种主动塑造自身信息工作空间的能力是其增强的鲁棒性与效率的关键，使其能够在复杂的长视界挑战中动态平衡对细粒度细节的需求与连贯的长期计划。
