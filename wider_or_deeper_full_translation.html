ੱ 摘要 (Abstract)

在推理时通过扩展计算来提升大语言模型（LLM）性能已成为一个关键的研究方向。现有方法主要遵循两种范式：重复采样（Repeated Sampling）——对同一问题多次采样以增加找到高质量解的机会；顺序改进（Sequential Refinement）——迭代地优化已有候选解。然而，这两种方法分别偏向于"更宽"或"更深"的搜索策略，且各自的最优性高度依赖于具体任务和模型特性。

本文提出自适应分支蒙特卡洛树搜索（Adaptive Branching MCTS, AB-MCTS），一种统一的框架，能够在"更宽"（生成新的独立候选）和"更深"（细化现有响应）之间动态切换。AB-MCTS 通过贝叶斯后验更新来自适应地决定每个节点的扩展方向，无需预设分支因子上限。我们提出两种变体：

  • AB-MCTS-M：基于混合效应模型（Mixed-Effects Model），利用马尔可夫链蒙特卡洛（MCMC）进行节点选择。
  • AB-MCTS-A：基于节点聚合（Node Aggregation），使用解析可处理的高斯或 Beta 后验分布。

在代码生成、数学推理、逻辑推理等多个基准测试上的实验表明，AB-MCTS 一致性地优于或匹敌现有的重复采样和顺序改进基线。此外，我们进一步将 AB-MCTS 扩展到多 LLM 场景（Multi-LLM AB-MCTS），通过 Thompson Sampling 在不同模型间动态分配搜索预算，在 ARC-AGI-2 基准上实现了超过 30% 的 Pass@k 成功率。

ੲ 引言 (Introduction)

近年来，大型语言模型（LLM）在各类任务上展现出卓越的能力。然而，如何在不重新训练模型的前提下进一步提升其性能，成为了一个核心研究课题。推理时扩展（Inference-Time Scaling）——即在推理阶段投入更多的计算资源——已被证明是一种有效的途径。

图 1：推理时扩展方法的四象限分类。横轴表示是否允许生成新候选（Wider vs. Deeper），纵轴表示是否利用历史信息指导搜索。AB-MCTS 统一了四个象限中的关键优势：既支持新候选生成又支持已有解的细化，同时通过贝叶斯后验实现自适应方向决策。

如图 1 所示，现有的推理时扩展方法可以按照两个维度进行分类：（1）是否生成全新的候选解（即"更宽" vs."更深"）；（2）是否利用历史搜索信息来指导下一步的决策。

左下象限：重复采样（Repeated Sampling）是最简单的方法——对同一问题多次调用 LLM 独立采样。虽然这种方法天然支持"更宽"的探索，但它完全忽略了各次采样之间的关系，无法利用已获得的反馈来引导后续搜索。

右下象限：顺序改进（Sequential Refinement / Self-Correction）则聚焦于"更深的"搜索路径——每次迭代都将当前最优解作为上下文输入，要求 LLM 对其进行修正或改进。这种方法能够逐步提升解的质量，但容易陷入局部最优，且缺乏对新方向的探索能力。

右上象限：蒙特卡洛树搜索（MCTS）及其变体（如标准 MCTS、Progressive Widening 等）同时具备"更宽"和"更深"的能力，并通过树结构维护搜索历史。然而，传统 MCTS 的分支因子通常是固定的或由超参数预先设定的，无法根据任务特性动态调整。

本文提出的 AB-MCTS 位于左上象限：它不仅统一了"更宽"和"更深"两种搜索模式，还通过贝叶斯后验推断实现了完全自适应的分支因子控制——算法在每个节点处自动判断应该继续深入细化还是横向扩展新候选，而无需人工设定任何固定宽度参数。

这一设计带来的核心优势包括：

  1. 无上限分支因子：充分利用 LLM 输出的多样性，不受预定义宽度约束的限制。
  2. 任务无关的自适应性：同一套算法在不同类型任务（代码、数学、逻辑等）上均能自动调整搜索策略。
  3. 理论保证：基于 Thompson Sampling 的框架提供了概率性的探索-利用权衡机制。

ੳ 相关工作 (Related Work)

3.1 推理时扩展方法概览

重复采样类方法：Best-of-N 采样是此类方法的代表，通过对同一提示词进行 N 次独立采样并选取最佳输出。近期工作如 s1 (Snell et al., 2025) 进一步证明了即使仅用少量精心策划的数据配合 Budget Forcing 技术，也能在推理阶段显著提升性能。这类方法的局限性在于缺乏对搜索历史的利用——每次采样都是独立的。

顺序改进类方法：Self-Refine (Huang et al., 2023)、STaR (Zelikman et al., 2022)、Reflexion (Shinn et al., 2023) 等方法通过迭代式自我纠正来提升答案质量。DeepConf (Meta AI) 利用置信度信号过滤低质量轨迹并实现早停。这些方法擅长深度优化但容易陷入局部最优。

树搜索方法：标准 MCTS 在 LLM 推理中的应用（如 MCTS-AHD, ReST-MCTS*, LiTS, TabTracer, MASTER 等）构建显式的搜索树来组织多个候选。Progressive Widening 通过动态调整分支因子来平衡探索与利用，但其调整规则仍依赖手动设置的超参数。我们的 AB-MCTS 与 Progressive Widening 最接近，但关键区别在于：AB-MCTS 使用数据驱动的贝叶斯后验推断而非固定的启发式规则来确定每一步的分支行为。

3.2 贝叶斯优化与 Thompson Sampling

Thompson Sampling (1933) 是一种经典的 bandit 算法，通过从后验分布中采样来实现探索-利用权衡。它在推荐系统、临床试验、在线广告等领域有广泛应用。近年来的研究表明，Thompson Sampling 在黑箱优化场景中往往优于 UCB 类方法。Bao (Marcus & Papaemmanouil, 2018) 将其应用于数据库查询优化中的 Hint 选择问题。我们的工作首次将 Thompson Sampling 的思想系统地引入到 LLM 推理时的树搜索框架中。

ੴ 方法 (Method)

4.1 预备知识 (Preliminaries)

考虑如下问题设定：给定一个待求解的问题 $q$ 和一个 LLM 答案生成器 $f_{\text{LLM}}$（可以是任意预训练的大语言模型），目标是在有限的生成预算 $N$ 内（以 LLM 调用次数度量），找到一个尽可能高质量的答案 $a^*$。

我们将答案生成建模为以下过程：

$$a = f_{\text{LLM}}(q, c) \tag{1}$$

其中 $c$ 表示可选的上下文信息（例如已有的候选解或中间步骤）。对于每次 LLM 调用，我们获得一个评分 $r \in \mathbb{R}$（由外部评价函数给出，如代码执行的 pass/fail、数学题的正确性等）。

MCTS 基础：标准 MCTS 维护一棵搜索树，其中每个节点代表一个部分解或完整解。在每一轮中，算法执行以下四步操作直到耗尽预算：

  1. 选择（Selection）：从根节点出发，按特定策略（通常基于 UCB 值）向下遍历至叶节点。
  2. 扩展（Expansion）：对选中的叶节点添加一个新的子节点（或若干子节点）。
  3. 模拟/评估（Simulation/Evaluation）：对新节点进行评估，获得奖励值。
  4. 回溯（Backpropagation）：将奖励值向上传播至根节点，更新沿途所有节点的统计量。

标准 MCTS 的关键局限在于：扩展步（Step 2）中的子节点数量（即分支因子 $w$）需要预先指定或由固定规则确定，无法根据实时观察到的奖励信息进行调整。

4.2 自适应分支 MCTS (Adaptive Branching MCTS)

我们的核心创新在于将 Step 2 替换为一个自适应的二元决策：在选中某个节点后，算法需要决定是：

  • GEN 动作：生成一个全新的独立候选解（对应"更宽"的方向）；
  • CONT 动作：基于当前节点的内容继续细化/改进（对应"更深"的方向）。

这个决策不是由固定规则决定的，而是通过贝叶斯后验推断——算法维护关于每种动作期望收益的后验分布，并在每一步通过 Thompson Sampling 从后验中进行采样来做出决策。

Algorithm 1: Adaptive Branching MCTS
─────────────────────────────────────
for iteration = 1, 2, ..., N {
// Step 1: 选择节点
node ← Select(root)
// Step 2: 自适应分支决策
action ← BranchDecision(node) // GEN 或 CONT
if action == GEN {
child ← f_LLM(q) // 生成全新候选
} else { // action == CONT
child ← f_LLM(q, node.content) // 细化当前解
}
// Step 3: 评估
r ← Evaluate(child)
// Step 4: 回溯 + 后验更新
Backpropagate(child, r)
UpdatePosterior(node, action, r)
}
return BestSolution(root)

4.3 AB-MCTS-M：混合模型方法 (AB-MCTS-M: Mixed-Effects Model)

AB-MCTS-M 将节点选择的建模视为一个混合效应模型（Mixed-Effects Model）问题。对于每个被选中的节点，我们观测到一个带噪声的评分：

$$\tilde{r}_N = \alpha_j + \sigma_y \tilde{\epsilon}_N \tag{2}$$ $$\alpha_j = \mu_\alpha + \sigma_\alpha \epsilon_j \tag{3}$$ $$\tilde{\epsilon}_N \sim \mathcal{N}(0, 1), \quad \epsilon_j \sim \mathcal{N}(0, 1) \tag{4}$$

其中 $\alpha_j$ 表示第 $j$ 组动作（GEN 或 CONT）的真实期望评分，$\mu_\alpha$ 是全局均值，$\sigma_\alpha$ 衡量组间变异。我们对参数设置共轭先验：

$$\mu_\alpha \sim \mathcal{N}(\mu_0, \sigma_0^2), \quad \sigma_\alpha^2 \sim \text{Inverse-Gamma}(a_0, b_0) \tag{5}$$

在每次观测到新的 $(j, r)$ 后，我们通过 MCMC（特别是 Gibbs 采样）更新后验分布，然后从后验中采样 $\alpha_j$ 来决定下一轮的动作选择。选择 GEN 还是 CONT 取决于哪个组的后验采样值更高。

图 2：AB-MCTS-M 示例搜索树与后验演化。左侧展示了一棵经过若干步搜索后的示例树，不同颜色表示不同的分支决策（GEN / CONT）。右侧展示了随着搜索进行，GEN 组和 CONT 组的后验分布演化过程——当某一组持续产生更高评分时，其后验均值上升、方差缩小，使得算法倾向于选择该方向。

4.4 AB-MCTS-A：节点聚合方法 (AB-MCTS-A: Node Aggregation)

AB-MCTS-M 虽然强大，但 MCMC 采样带来了一定的计算开销。AB-MCTS-A 提供了一种轻量级替代方案，它将搜索树的每一层抽象为两种类型的节点：

  • GEN 节点：触发全新候选生成的节点。当一个 GEN 节点被选中时，调用 $f_{\text{LLM}}(q)$ 生成新候选。
  • CONT 节点：触发细化操作的节点。当选中一个 CONT 节点时，调用 $f_{\text{LLM}}(q, c)$ 对父节点的候选进行改进。
图 3：AB-MCTS-A 的搜索树结构与节点聚合示意。上方为 CONT（顺序改进）主导的深层窄树，下方为 GEN+CONT 自适应混合的较宽树。AB-MCTS-A 在每个层级都面临 GEN vs. CONT 的二元选择（由 Thompson Sampling 决定），从而形成灵活的树形结构。图中同时展示了高斯/Beta 后验的更新方式。

AB-MCTS-A 为每个节点维护独立的先验分布，并根据评分指标的类型采用不同的共轭模型：

情况一：评分无界（如连续型准确率分数）→ 高斯模型

$$\mu_l \sim \mathcal{N}(\mu_0, \sigma_0^2), \quad \sigma_l^2 \sim \text{Inverse-Gamma}(a_0, b_0) \tag{6}$$ $$\mu_l | \mathbf{r}_l \sim \mathcal{N}\left(\frac{n_l \bar{r}_l / \sigma_l^2 + \mu_0/\sigma_0^2}{n_l/\sigma_l^2 + 1/\sigma_0^2}, \left(n_l/\sigma_l^2 + 1/\sigma_0^2\right)^{-1}\right) \tag{7}$$

情况二：评分有界于 [0,1]（如 Pass@k 率）→ Beta 模型

$$\theta_l \sim \text{Beta}(\alpha_0, \beta_0) \tag{8}$$ $$\theta_l | \mathbf{r}_l \sim \text{Beta}\left(\alpha_0 + S_l, \beta_0 + n_l - S_l\right) \tag{9}$$

其中 $n_l$ 是第 $l$ 个节点的访问次数，$\bar{r}_l$ 是平均评分，$S_l$ 是成功次数总和。

在每次选择时，AB-MCTS-A 从每个活跃节点的后验分布中采样，选择采样值最大的节点进行扩展。如果被选中的是 GEN 节点则生成新候选，如果是 CONT 节点则细化现有解。

ੵ 实验 (Experiments)

5.1 实验设置 (Experimental Setup)

基准测试：我们在以下三类任务上评估 AB-MCTS：

  1. 代码生成：LiveCodeBench（竞争编程）、CodeContest（竞赛编程）
  2. 逻辑推理：MLE-Bench（三个机器学习竞赛任务：Nomad2018、Spooky Author Identification、Random Acts of Pizza）
  3. 抽象推理：ARC-AGI（Abstraction and Reasoning Corpus）

基础模型：主要使用 GPT-4o 作为骨干 LLM，同时在附录中报告 DeepSeek-V3、o4-mini、Gemini-2.5 Pro、DeepSeek-R1 等模型的结果。

基线方法：

  • Repeated Sampling：对同一问题进行 N 次独立采样，取最优解。
  • Sequential Refinement：每次将当前最优解作为上下文传入 LLM 要求改进。
  • Standard MCTS：使用固定分支因子 $w=5$ 的标准 MCTS。
  • Progressive Widening (PW)：带渐进式分支因子的 MCTS 变体。

评估指标：对于代码和逻辑任务使用 Pass@1（第一次尝试的正确率）；对于 ARC-AGI 使用官方标准的 Pass@2（两次提交中至少一次正确）。生成预算 N 根据任务复杂度设置为 27–512 不等。

MLE-Bench 详细结果（Table 2）：在三个 MLE-Bench 子竞赛上，AB-MCTS-M 表现尤为突出。值得注意的是，最优基线方法在不同竞赛之间存在显著变化——这恰好验证了自适应方法的必要性。例如：

  • Nomad2018：Sequential Refinement 最终取得最佳分数，而 Repeated Sampling 无任何提升。
  • Spooky Author Identification：Standard MCTS 在整个预算范围内持续改进。
  • Random Acts of Pizza：Repeated Sampling 显著优于其他所有基线。

尽管基线表现高度可变，AB-MCTS-M 在所有三个竞赛上始终交付强劲的性能，凸显了其鲁棒性和适应性。

图 6（上部区域）：ARC-AGI 任务上扩展至预算 N=512 的性能曲线。当预算大幅增加时，AB-MCTS-A (Gaussian) 和 AB-MCTS-A (Beta) 的优势更加明显，表明自适应搜索在大规模计算预算下的价值。

搜索树可视化分析（Figure 7）：图 7 展示了 AB-MCTS-M 在 MLE-Bench 的 Random Acts of Pizza 任务上实际生成的一棵搜索树。仔细观察可以发现：

  • 树的某些分支呈现出明显的链状结构（连续的 CONT 操作），说明算法在这些方向上发现了值得深入挖掘的候选。
  • 另一些节点产生了多个子节点（GEN 操作），表明算法在那些时刻判断需要探索全新的方向。
  • 整体形态既非纯粹的宽树也非纯粹的深树，而是根据搜索过程中收集到的奖励信号自然形成的混合结构。

ARC-AGI 扩展分析（Figure 6）：我们将生成预算扩展到 N=512 以考察 AB-MCTS 在大规模计算条件下的表现。结果显示：

  • AB-MCTS-A (Beta) 在 N=512 时达到约 18% 的 Pass@2 率，比 Repeated Sampling 高出约 3 个百分点。
  • 更重要的是，AB-MCTS 方法的增长斜率始终高于基线方法——这意味着随着预算进一步增大，优势有望继续扩大。
  • Sequential Refinement 在 ARC-AGI 上表现不佳（Pass@2 仅约 8.7%），这是因为该任务需要多样化的尝试空间，单纯的深度改进容易陷入错误模式的循环强化。

੶ 结论 (Conclusion)

本文提出了 AB-MCTS——一种用于 LLM 推理时扩展的自适应分支蒙特卡洛树搜索框架。通过引入贝叶斯后验推断来驱动"更宽"与"更深"之间的动态决策，AB-MCTS 克服了现有方法需要预设分支因子的限制。我们在代码生成、逻辑推理和抽象推理三类任务上进行了全面的实验验证，结果表明 AB-MCTS 一致性地匹敌或超越了重复采样、顺序改进和标准 MCTS 等强基线。

局限性：当前工作主要集中在单 LLM 场景。虽然我们在附录 D 中初步探讨了多 LLM 协作搜索，但更复杂的模型组合策略（如针对不同子任务动态选择专门化模型）仍有待深入研究。此外，后验更新的计算开销（尤其是 AB-MCTS-M 的 MCMC 部分）在极端延迟敏感的场景下可能需要近似加速方案。

未来方向：(1) 引入难度估计机制进一步增强自适应性；(2) 探索与其他推理时增强技术（如过程奖励模型、置信度过滤）的结合；(3) 将 AB-MCTS 应用于更多领域（如科学发现、多模态推理等）。(4) 推进 LLM 推理时扩展的 Pareto 前沿。

§ 附录 A：技术细节 (Appendix A)

A.1 问题形式化定义

我们将 LLM 推理时的答案搜索问题形式化定义如下：

输入：问题 $q$，LLM 答案生成器 $f_{\text{LLM}} : (\mathcal{Q} \times \mathcal{C}) \to \mathcal{A}$，评价函数 $g: \mathcal{A} \to \mathbb{R}$，预算 $N \in \mathbb{N}^+$。

输出：最优答案 $a^* = \arg\max_{a \in \mathcal{S}} g(a)$，其中 $\mathcal{S}$ 是在预算内生成的答案集合。

约束：总共最多调用 $f_{\text{LLM}}$ 次 $N$ 次。

A.2 与 Progressive Widening 的详细对比

Progressive Widening (Chaslot et al., 2008) 是最接近 AB-MCTS 的前人工作。两者都试图动态调整分支因子，但存在本质区别：

维度Progressive WideningAB-MCTS (Ours)
决策依据固定公式: $n(v)^\alpha \cdot C$（需调超参 $k, \alpha$）贝叶斯后验采样（数据驱动，无需调参）
信息利用仅用访问次数 $n(v)$完整利用评分序列 $\{r_i\}$
理论保证经验性启发式Thompson Sampling 渐近最优性
分支上限有上限（由公式隐式限制）无上限（充分利用 LLM 多样性）

表 4（附录 C.4）展示了在 LiveCodeBench 上与 Progressive Widening 的直接比较结果。当 PW 的超参数设置适当时 $(k, \alpha) = (5, 0.5)$，其性能与 AB-MCTS 可比；但设置不当时 $(k, \alpha) = (1, 0.45)$ 会产生最差结果，且 $(10, 0.55)$ 导致高方差和搜索不稳定性。相比之下，AB-MCTS 无需任何超参数调整即可保持稳健性能。

A.3 混合效应模型的细节

AB-MCTS-M 中的混合效应模型（方程 2-5）可以理解为一种层次贝叶斯模型：

  • 第一层（观测层）：每个节点的评分 $\tilde{r}_N$ 是其所属组真实期望 $\alpha_j$ 加上噪声。
  • 第二层（组层）：每组期望 $\alpha_j$ 围绕全局均值 $\mu_\alpha$ 波动，波动幅度由 $\sigma_\alpha$ 控制。
  • 先验层：$\mu_\alpha$ 和 $\sigma_\alpha^2$ 分别设为正态和逆伽马先验，确保后验共轭可解。

MCMC 更新过程采用 Gibbs 采样器，每次迭代交替采样各层参数的条件分布。由于我们的场景中观测数据量相对较小（通常 N < 500），MCMC 在几步内即可收敛到合理的后验估计。

A.4 参数更新规则汇总

AB-MCTS-A 后验更新规则速查：
• 高斯模型（连续评分）：后验均值是先验均值与样本均值的加权平均，权重反比于方差。
• Beta 模型（[0,1] 有界评分）：$\alpha_{post} = \alpha_0 + S_l$, $\beta_{post} = \beta_0 + n_l - S_l$，其中 $S_l$ 为累计成功次数。
• Thompson Sampling 决策：从每个活跃节点的后验中采一个值，选最大的节点扩展。

A.5 搜索走查示例

以下以 LiveCodeBench 上的一道编程题为例，展示 AB-MCTS-M 前 5 步的典型搜索过程：

  1. Step 1: 根节点 → GEN → 生成候选 A（评分 0.3）。后验: GEN 组均值上升。
  2. Step 2: 根节点 → GEN（TS 选择 GEN 因其后验更高）→ 生成候选 B（评分 0.5）。
  3. Step 3: 候选 B 节点 → CONT（TS 判断细化 B 更有潜力）→ 生成 B'（评分 0.7）。
  4. Step 4: 候选 B' 节点 → CONT → 生成 B''（评分 0.65）。CONT 后验持续增强。
  5. Step 5: 根节点 → GEN（GEN 后验因 Step 1-2 的数据仍具竞争力）→ 生成候选 C（评分 0.4）。

这个走查展示了算法如何在"深入挖掘有潜力的候选"（Step 3-4 对 B 的细化）和"探索全新方向"（Step 1-2, 5 的新 GEN）之间灵活切换。

A.6 超参数敏感性分析

我们测试了 Standard MCTS 使用不同固定分支因子 $w \in \{3, 5, 10\}$ 时的性能（Table 6, 附录 C.8），使用 DeepSeek-V3 在 LiveCodeBench 上：

Table 6 固定分支因子消融实验结果：
• w = 3: Pass@1 = 42.9 ± 1.8%

  • 有严格时间约束时 → AB-MCTS-A：高斯/Beta 后验更新是解析性的（无需迭代），显著更快。对于延迟敏感的应用是更好的选择。
  • 需要更宽泛探索的任务 → AB-MCTS-A：由于其在每一层都面临 GEN/CONT 二选一（连续 d 次 CONT 概率呈几何衰减），天然倾向于生成更宽的树。ARC-AGI 就是此类任务的典型代表。

C.9 AB-MCTS 性能鲁棒性讨论

在其他基线方法中，分支因子要么是预定义的，要么由超参数决定。例如 Standard MCTS 需要预先指定分支因子 $w$。然而，宽度 vs 深度的效率强烈依赖于任务类型和 LLM 本身。这一点在 Table 1 和 Table 2 中得到充分体现：AB-MCTS 在各类任务上表现出色，而其他方法在某些任务上表现优异但在另一些上表现不佳。这归功于 AB-MCTS 的自适应分支本质——算法根据观测到的奖励自动调整宽或深的方向。

这在 LLM 推理时扩展的上下文中尤其有价值：近年来 LLM 被用于解决各种类型的任务（数学、代码等），一种开箱即用且适用于多种任务类型的搜索算法具有很高的需求。

C.10 推进 Pareto 前沿的两个方向

我们提出两个潜在的改进方向以推进 LLM 推理时扩展的 Pareto 前沿：

  1. 通过难度估计增强自适应性：从收集到的奖励中显式估计问题难度。对于困难问题（低奖励标识），策略将动态切换——例如从深度搜索（AB-MCTS-M）切换到广度搜索（AB-MCTS-A）。这受到最优搜索策略依赖难度的研究发现的启发 [27]。
  2. 多 LLM 协作搜索：利用多个 LLM 的不同优势进行单次搜索内的协作。这通过扩展 AB-MCTS 引入多个 GEN 节点实现（每个 LLM 一个）。详见附录 D。

§ 附录 D：多 LLM AB-MCTS (Multi-LLM Extension)

D.1 动机与方法概述

根据任务的不同，使用超过一个 LLM 来搜索答案可能带来优势。例如，如果 LLM A 能生成更多样化的初始答案而 LLM B 擅长细化改进，那么同时使用两者来构建答案树有望提升性能。本节展示如何将 AB-MCTS 扩展到多 LLM 场景，并报告在 ARC-AGI-2 基准上的实验结果。

D.1.1 多个 LLM 作为答案生成器

假设有 $L$ 个 LLM 可用于答案生成。记第 $l$ 个 LLM 实现的答案生成为 $f_{\text{LLM}}^l$，$l = 1, \ldots, L$。整体流程与单 LLM AB-MCTS（Algorithm 1）相同，增加一步：在每次扩展时从 $L$ 个可用生成器中选择一个用于节点扩展。选择过程使用当前答案树 $T$ 的状态信息。

D.1.2 生成器选择算法 I：单一 GEN 节点

在此算法中，节点选择与标准 AB-MCTS 相同，随后附加一个额外的生成器选择步骤。对于每个生成器 $l$，我们获得它生成的所有节点的集合 $\mathcal{N}_l \subseteq T$（若该生成器尚未被选中则集合为空）。$\mathcal{N}_l$ 中节点的评分用于计算生成器 $l$ 将要生成的新节点的期望评分的后验分布。对所有 $L$ 个后验完成计算后，应用 Thompson Sampling：从每个分布中采样一个值，选择采样值最高的生成器进行扩展。

后验建模方式与 AB-MCTS-M/A 中的节点目标选择完全相同，只是将"节点索引"替换为"生成器索引"。

D.1.3 生成器选择算法 II：多个 GEN 节点

另一种方案是在树中每个节点下附加多个 GEN 子节点——每个可用生成器一个。在每个节点的扩展目标选择过程中执行以下操作：

  1. 对与每个生成器关联的子树独立应用 AB-MCTS 选择逻辑。
  2. 使用 Thompson Sampling 确定每个生成器的最佳节点（GEN 节点或已有子节点）。
  3. 比较 $L$ 个最佳节点的分数，选择总体分数最高的那个。

此算法旨在更好地捕获搜索树的局部上下文信息，允许在求解过程的每个阶段自适应地选择最合适的生成器。

更重要的是，我们观察到单个模型无法解决的问题通过多模型协作得到了解决——这表明存在超越简单"最佳模型匹配"的协同交互效应。

图 15：Multi-LLM AB-MCTS-A 在一道成功解决的 ARC-AGI-2 问题上的示例搜索树。每个节点标注了生成步骤编号，颜色表示选择的 LLM（黄色 = o4-mini，绿色 = Gemini-2.5 Pro，紫色 = DeepSeek-R1-0528）。黄色节点生成了正确解决测试用例的代码。这个问题在单独使用任何一个模型时都无法解决。
图 16：模型协作示意图（对应 Figure 15 中的问题 36a08778）。展示了 o4-mini 生成的错误中间解如何作为有用的提示被 DeepSeek-R1-0528 利用，进而产生正确的最终方案。R1 分析了 o4-mini 方案中的两个主要缺陷：（1）水平扩散时的单元格着色遗漏；（2）全局 visited 集导致的处理冲突。修正这些问题后成功解决了测试用例。

图 15 和图 16 共同描绘了一个完整的跨模型协作案例：o4-mini 生成了一个不完美的中间解（虽然不正确但包含有用信息），DeepSeek-R1-0528 分析了其中的缺陷并进行了针对性修正，最终 Gemini-2.5 Pro 或 R1 产出了正确的解决方案。这一结果表明 Multi-LLM AB-MCTS 能够促进异构前沿 LLM 之间灵活有效的协作。

D.3 挑战与未来工作

虽然我们的主要评估聚焦于搜索能力（Pass@k 指标），我们也基于 Pass@2 标准做了初步参考评估。使用简单的基于规则的方法来选择两个最终答案（优先选择搜索后期产生的高奖励代码），Multi-LLM AB-MCTS 达到了 19.2% 的 Pass@2。虽然这是一个有前景的结果，但与 30% 的 Pass@250 率相比仍有超过 10 个百分点的差距。未来工作应致力于缩小这一差距，开发更复杂的最终答案选择算法。潜在方向包括构建更精确的奖励模型或集成 LLM-as-a-Judge 以实现对候选方案的更细致评估。
