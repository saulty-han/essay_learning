1 研究动机：多模态表示能给广告系统带来什么

淘宝展示广告主流采用离散 ID 特征（item id / user id 等），无法捕获商品图像、文案等多模态语义，且对长尾 / 新物品存在严重冷启动问题。已有多模态方案尝试端到端训练，但工业落地中预训练编码器、新物品实时表示、系统升级的算力成本常不抵收益。

ID 特征的固有缺陷

离散 ID 特征难以表达物品的视觉 / 文本语义，长尾与全新物品的 ID 缺乏历史交互，导致表征稀疏、冷启动差。系统对多模态信号的利用长期停留在"低垂果实"之外。

端到端多模态的成本困境

直接端到端训练多模态编码器在工业级流量下算力过高；预训练编码器、新物品实时表示、系统升级三方面的部署成本，往往使离线收益无法转化为线上增益。

本文主张：两阶段低成本落地

提出"预训练多模态表示（SCL）+ 与 ID 模型集成（SimTier + MAKE）"的两阶段框架：先在离线侧用对比学习得到高质量多模态表示，再在在线侧以极低成本把表示接入已有 ID 模型，化解"训练成本 vs 线上收益"的矛盾，并显著缓解冷启动。

2 框架概览：两阶段多模态表示框架

框架分两阶段：① 离线预训练多模态表示（SCL 语义对比学习）；② 将其集成进 ID 模型（SimTier 相似度向量 + MAKE 知识向量）。生产系统中，新物品实时经编码器生成表示，经多模态索引表进入近线训练与在线推理，端到端延迟仅数秒。

端到端流程

离线 SCL 预训练得到物品多模态表示 → SimTier 生成目标物品与历史交互的相似度向量、MAKE 提取多模态知识向量 → 二者与 ID embedding 拼接进 MLP 做 CTR 预估；在线侧新物品实时经编码器生成表示，进入近线训练与在线推理。

3 阶段一：多模态表示预训练（SCL）

SCL（Semantic Contrastive Learning）用"用户搜索 query + 购买物品"构造语义相似对，通过对比学习把图、文物品映射到同一语义空间。

难负样本挖掘

用 MoCo 大记忆库（K=196800）扩大负样本规模；Triplet Loss 主动挖掘难负样本（hard negatives）；InfoNCE 作为对比损失。这种"大库 + 难负"组合是表示质量的关键。

预训练数据
"query + 购买物品"语义对 + 点击难负样本；下游 CTR 用淘宝一周曝光日志。
效果
SCL 的 Acc@1 = 0.7474，远超 CLIP-O 的 0.2559；且 Acc@N 与下游 GAUC 强相关，可作快速迭代的代理指标。

4 阶段二：与 ID 模型集成（SimTier + MAKE）

把多模态表示接入 ID 模型有两个难点：直接拼接联合训练会导致多模态参数学习不充分；ID 与多模态所需训练 epoch 数不同。SimTier 与 MAKE 分别用"简化"与"解耦"两种思路解决。

SimTier：相似度向量

计算目标物品与历史交互物品的语义相似度，分 N 档统计成 N 维向量拼接输入。它把多模态信号"简化"为轻量特征，避免与 ID 联合训练不充分的问题。

MAKE：多模态知识向量

用 DIN + 4 层 MLP 对多模态表示做多 epoch 预训练（CTR 预训练），把输出 vMAKE 接入下游，解耦了 ID 与多模态的 epoch 差异——ID 一 epoch 即过拟合，多模态需多 epoch。

5 工业部署与冷启动策略

实时表示与低成本落地

生产系统中，新物品实时经预训练编码器生成多模态表示，写入多模态索引表，供给近线训练与在线推理，端到端延迟仅数秒。相比端到端训练，本框架把多模态收益以极低成本带入线上。

冷启动缓解

多模态表示自带语义，新物品无需历史交互即可获得有效表征，低频 / 新广告的 CTR 提升尤为显著（新广告 CTR +6.9%），直接缓解 ID 特征的冷启动困境。

6 两阶段框架的端到端整合

离线 / 在线协同

离线侧 SCL 预训练产出物品表示；SimTier 生成相似度向量、MAKE 提取知识向量，与 ID embedding 拼接进 MLP。在线侧新物品实时生成表示供训练 / 推理。两阶段非端到端，但通过"解耦 epoch + 简化特征"兼顾了训练充分性与系统成本。

部署成本
无需改动主模型结构，仅新增表示生成与索引表；线上延迟仅数秒，算力开销可控。

7 实验评估：预训练质量与线上增益

设置与基线

预训练数据用 query + 购买 + 点击难负；CTR 数据用淘宝一周曝光日志。基线：CLIP-O / CLIP-E / ID / Vector / SimScore。下游用 DIN 结构。

主结果：SCL 的 Acc@1 = 0.7474，远超 CLIP-O 的 0.2559；CTR 上 SimTier + MAKE 使 GAUC +1.25%、AUC +0.75%，低频物品提升更显著。线上：CTR +3.5%、RPM +1.5%、ROI +2.9%，新广告 CTR +6.9%。

8 敏感性分析：消融与代理指标

SCL 组件消融

去掉 Triplet 后 Acc@1 降至 0.6957，再去掉 MoCo 降至 0.5760，证明难负挖掘与大记忆库对表示质量至关重要。MAKE 的 epoch 越多效果越好。

代理指标

Acc@N 与下游 GAUC 强相关，可作为预训练质量的快速代理指标，显著加速迭代；低频组 AUC 提升幅度大于高频组。

9 代码使用与实战（含 GitHub 源码、安装命令、使用示例）

论文与代码

本文为阿里巴巴集团发表于 CIKM 2024（arXiv:2407.19467，ACM DOI 10.1145/3627673.3680068）。文中未提供公开代码（阿里内部系统）。以下为按文中方法的可复现说明。

# 1) 预训练多模态表示（SCL）：构建 query+购买 语义对
python scl_pretrain.py --encoder clip --memory_bank 196800 \
    --loss "triplet+infonce" --data query_buy_pairs.jsonl

# 2) 训练 SimTier：目标物品与历史交互的相似度分档向量
python train_simtier.py --repr scl.pt --n_bins 8

# 3) 训练 MAKE：DIN+4层MLP 多 epoch 预训练多模态知识向量
python train_make.py --repr scl.pt --epochs 12 --backbone din

# 4) 集成进 ID 模型（拼接 SimTier + MAKE + ID embedding 进 MLP）
python train_ctr.py --id_emb id.npy --simtier simtier.pt --make make.pt

复现要点：SCL 的关键是大记忆库 MoCo + Triplet 难负；SimTier 用"简化"避免联合训练不充分，MAKE 用"解耦 epoch"解决 ID 与多模态的 epoch 矛盾。下游可用 DIN 结构，Acc@N 作预训练代理指标。

10 总结与未来展望

本文提出两阶段多模态表示框架：SCL 用对比学习（MoCo + Triplet）得到高质量物品表示，SimTier 与 MAKE 以"简化 / 解耦"两种方式低成本接入 ID 模型，在淘宝展示广告上实现 CTR +3.5%、新广告 CTR +6.9% 的线上增益并缓解冷启动。

Q1：为什么不直接拼接多模态向量到 ID 模型？
联合训练时多模态参数学习不充分（ID 一 epoch 就过拟合），需用 SimTier（简化特征）或 MAKE（解耦多 epoch 预训练）分别处理。
Q2：ID 与多模态的 epoch 矛盾怎么解？
ID 特征一 epoch 即过拟合，多模态需多 epoch；MAKE 单独对多模态做多 epoch 预训练再接入，解耦二者差异。
Q3：如何快速评估预训练质量？
用 Acc@N 作为代理指标，它与下游 GAUC 强相关，可大幅加速预训练迭代，无需每次都跑全量 CTR。

局限与未来

两阶段非端到端；难负构造等细节未深入；仅在淘宝场景验证。未来可探索端到端联合训练与跨场景（如短视频广告）迁移。
