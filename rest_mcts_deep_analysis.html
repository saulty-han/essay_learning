1 研究动机：最终答案对 ≠ 过程对

LLM 自训练普遍用"生成 → 筛正确最终答案"做训练数据，但"最终答案对 ≠ 过程对"——轨迹中常有错误 / 无用中间步骤，构成大量假阳性样本，限制微调质量。训练逐步骤的过程奖励模型（PRM）需稠密人工标注，不可扩展。

假阳性轨迹问题

仅靠最终答案筛出的"正确"轨迹常含错误中间步，用其微调会污染策略模型。需要能反映"过程正确性"的信号。

PRM 标注瓶颈

逐步骤过程奖励模型需要稠密人工标注，成本高、不可扩展。需要自动构造过程奖励的方法。

本文主张：过程奖励引导的树搜索

ReST-MCTS* 用改进的 MCTS* 在推理树上反向传播推断每步正确性，自动构造过程奖励并收集高质量训练轨迹，实现策略模型与奖励模型的互增强（"MuZero 式"学习）。

2 框架概览：MCTS* + 互自训练

四大组件：MCTS*（树搜索策略）、PRM / 质量价值模型 Vθ（引导搜索）、Policy Model（生成步骤）、Self-Training（用搜索轨迹训策略与奖励模型）。二者相互促进。

互自训练循环

初始 SFT 策略 πS0、由 D0 经过程奖励推断构建 DV0 训 V0；每轮 i：πSi−1 + Vᵢ−1 引导 MCTS* 生成 DG 解 → 选正确解训 πSi → 提取价值数据训 Vᵢ。策略与奖励模型相互促进。

3 过程奖励引导的 MCTS*

MCTS* 以质量价值 vₖ 为目标，含节点选择、思维扩展、贪心 MC rollout、价值反传四阶段；无需完整生成即可在中间节点反传，并引入特殊 rollout 与 self-critic 提效。

质量价值与加权奖励

定义部分解质量值 vₖ（反映到正确答案的概率与步贡献），引入推理距离 mₖ 作权重：wₛₖ = (1−vₖ₋₁)/(mₖ+1)·(1−2rₛₖ)，vₖ = max(vₖ₋₁+wₛₖ, 0)，满足有界性定理 vₖ∈[0,1]。

4 树搜索回溯推断正确性

中间节点反传

用 MCTS* 在剪枝树上沿正确路径推断每节点 vₖ / rₛₖ 作为价值模型训练目标。与 ORM 不同，vₖ 可在中间节点反传，无需完整生成即可获得过程奖励，省算力且避免虚假正确性。

与 MATH-SHEPHERD 的区别

MS 用随机 rollout 合成奖励；本文用树搜索充分 rollout 推断每步能否导向正确答案，验证器精度更高。

5 自训练数据收集

互自训练管道

用 MCTS* 在剪枝树上沿正确路径推断每节点 vₖ/rₛₖ 作价值模型训练目标，同时筛正确轨迹做 SFT 训策略模型，迭代互补。正确解用于训策略、过程奖励用于训价值，二者同轮产出。

6 搜索与训练的整合

端到端数据流

每轮 i：当前策略 + 价值模型引导 MCTS* 生成解 → 选正确解训策略模型 → 提取价值数据训价值模型。策略模型质量提升 → 搜索生成更高质轨迹 → 价值模型更准 → 进一步反哺策略，形成互增强闭环。

7 实验评估：MATH / GSM8K / SciBench

设置与基线

数据集 MATH / GSM8K / TheoremQA（生成）、SciBench / SciEval（评测）；3 基座（LLaMA-3-8B-Instruct、Mistral-7B:MetaMATH、SciGLM-6B）。基线 ReSTEM、Self-Rewarding、CoT、ToT、SC、BoN、MATH-SHEPHERD。

主结果：SciGLM-6B 2 轮后平均 35.90 vs ReSTEM 33.27 / Self-Rewarding 33.56；MATH500 验证器 SC+ReST-MCTS* 39.0 > MS 37.3；SciBench 上 GLM4 16.77 vs ToT 15.82 / CoT 12.68；MATH 上 Iter#2 最高 48.5% 超 SC 42.5%。

9 代码使用与实战（含 GitHub 源码、安装命令、使用示例）

官方仓库

代码仓库：https://github.com/THUDM/ReST-MCTS（清华 KEG & Caltech，NeurIPS 2024，arXiv:2406.03816）。含数据 / 脚本。

# 1) 克隆仓库
git clone https://github.com/THUDM/ReST-MCTS.git
cd ReST-MCTS

# 2) 安装依赖
pip install -r requirements.txt

# 3) 用 MCTS* 生成高质量训练轨迹（自动构造过程奖励）
python mcts_search.py --model LLaMA-3-8B-Instruct --dataset MATH

# 4) 互自训练：SFT 策略 + 训价值模型
python self_training.py --policy_ckpt policy.pt --value_ckpt value.pt --rounds 2

复现要点：MCTS* 在剪枝树上沿正确路径反传 vₖ 作价值目标；正确轨迹训策略、过程奖励训价值，二者同轮产出、互增强。需真实答案（oracle）作为搜索终止信号。

10 总结与未来展望

ReST-MCTS* 用过程奖励引导的 MCTS* 自动构造高质量推理轨迹：在中间节点反传质量价值避免虚假正确性，并用互自训练让策略模型与价值模型相互促进，在 MATH / GSM8K / SciBench 上全面超越 ReSTEM / Self-Rewarding / MATH-SHEPHERD。

Q1：为何不直接用 ORM？
ORM 需完整生成才能反传代价高；vₖ 可在中间节点反传，省算力且避免假阳性样本。
Q2：过程奖励从哪来？
由 MCTS* 充分 rollout 推断每步能否导向正确答案，无需人工标注。
Q3：与 MATH-SHEPHERD 区别？
MS 用随机 rollout 合成奖励；本文用树搜索推断，验证器精度更高。

局限与未来

需真实答案（oracle）；未验证数学以外任务（代码 / agent / 对话 / SWE）；价值模型需扩展、数据过滤待改进。
