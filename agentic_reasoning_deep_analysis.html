🧱 基础层 — Foundational Agentic Reasoning

建立智能体在稳定环境中的核心能力：规划（Plan→Act, Tree of Thoughts）、工具使用（API调用、代码执行）、搜索（DFS/BFS/MCTS在推理空间中的搜索）。这是智能体推理的"地基"，所有高级能力都建立在基础能力之上。

🔄 进化层 — Self-Evolving Agentic Reasoning

智能体在动态变化的环境中通过反馈循环（外部评估信号）、记忆系统（情景记忆+语义记忆）和自省机制（Reflexion, Self-Refine）持续自我完善。关键在于从经验中学习并泛化到新场景。

多智能体系统通过角色分工（如 CAMEL 的角色扮演）、知识共享（共享记忆池或消息传递）和共识机制（辩论、投票）实现集体智能。MetaGPT、AutoGen 等框架使多智能体协作成为可能。

0 核心问题与研究动机

大语言模型（LLM）在封闭世界的数学和代码推理基准上展现了惊人的能力，但当这些模型被置于开放、动态的真实环境中时，它们的表现往往令人失望。原因在于：推理（Reasoning）和行动（Action）之间存在鸿沟。传统的 LLM 推理是一种"闭门造车"——模型在内部生成思维链，但无法与外部世界交互、获取反馈或修正错误。

智能体推理（Agentic Reasoning）的核心洞察是：真正的智能不仅需要"想"（Think），更需要"做"（Act）和"学"（Learn）。当一个 LLM 被赋予感知环境、调用工具、获取反馈的能力时，它就从一个静态的推理器变成了一个动态的智能体。这种范式转换带来的不仅是能力的扩展，更是问题解决方式的根本性变革——从单次推理到交互式探索，从静态知识到动态适应，从孤立决策到协作规划。

🔑 核心问题

如何在 LLM 已有的推理能力基础上，系统性地构建规划、行动、学习的能力，使 LLM 成为能在开放环境中自主解决问题的智能体？更进一步，如何让多个智能体协同工作，实现集体智能？

本综述的价值在于：在智能体推理领域爆发式增长的当下（2024-2026年间相关论文数量呈指数级增长），缺乏一个统一的分类框架来理解各种方法之间的关系和差异。本论文提供了这样的框架——三个互补维度，覆盖了从单智能体到多智能体、从无训练到训练优化、从理论到实践的完整图谱。

1 背景知识与前置概念

在深入智能体推理之前，我们需要理解几个关键的前置概念——这些概念构成了本综述讨论的基础，但并非本文首创，而是整个领域的共同知识基础。

LLM Reasoning → Agentic Reasoning 的范式转换

传统 LLM 推理（如 Chain-of-Thought、Tree of Thoughts）在封闭世界中运行：给定问题，模型生成推理链并给出答案。Agentic Reasoning 的关键区别在于引入了三个核心要素：(1) 环境交互——智能体可以观察环境状态、执行动作、获取反馈；(2) 工具使用——智能体可以调用外部工具（搜索引擎、代码解释器、数据库等）来扩展自身能力边界；(3) 迭代改进——智能体不是一次性给出答案，而是通过多轮交互持续优化策略。简言之，从"想一次"到"边做边想边改"。

ReAct：推理-行动融合的开创性工作

ReAct（Yao et al., 2023）是智能体推理的奠基性工作之一，它首次系统性地将推理（Reasoning）和行动（Acting）交织在同一个 LLM 生成过程中。在 ReAct 框架下，LLM 交替生成"思考"（Thought）和"行动"（Action）步骤：思考步骤负责推理和规划，行动步骤负责与环境交互。这种交织方式使得智能体能够基于观察结果动态调整推理方向，而非盲目执行预设计划。本综述中的大量方法都可以视为 ReAct 思想的扩展和深化。

In-Context Reasoning vs. Post-Training Reasoning

这是本综述区分的两种优化路径。In-Context Reasoning指在推理时通过精心设计的提示词、工作流编排和工具增强来提升智能体表现，无需修改模型参数——典型方法包括 CoT、ToT、ReAct 等。Post-Training Reasoning则通过强化学习（RL）、监督微调（SFT）或直接偏好优化（DPO）来训练模型学习更优的智能体行为——例如 Toolformer 训练模型学习何时调用工具，AgentTune 通过轨迹微调提升智能体规划能力。两条路径可以互补：先训练提升基础能力，再通过上下文编排进一步优化。

规划、搜索与决策的关系

在智能体推理中，规划（Planning）是将目标分解为可执行步骤的过程；搜索（Search）是在可能的行动空间中系统性地探索以找到最优路径；决策（Decision Making）是在特定状态下选择最佳行动。三者形成层次关系：规划提供全局蓝图，搜索验证和优化蓝图，决策在每一步执行最优选择。MCTS（蒙特卡洛树搜索）是连接三者的经典算法——它通过模拟来评估行动价值（搜索），构建最优行动序列（规划），并在每个节点选择最 promising 的分支（决策）。

2 方法论：三大维度详解

本综述的核心贡献在于提出了一个三维分类框架，将纷繁复杂的智能体推理方法纳入统一体系。下面逐一深入解析这三个维度。

2.1 基础层：Foundational Agentic Reasoning

基础层关注智能体在稳定环境中的核心单智能体能力。这是智能体推理的"地基"，包含三大支柱：

🗺️ 规划 (Planning)

规划是将复杂目标分解为可执行步骤的核心能力。关键方法包括：

• Plan-and-Solve：先生成完整计划再逐步执行，适合任务结构清晰的场景

• Adaptive Planning：根据环境反馈动态调整计划，适合不确定性高的场景

• Tree of Thoughts (ToT)：将推理过程建模为搜索树，在思维空间中系统性地探索多种可能

• Decomposition：将大任务分解为小任务（Task Decomposition），如 HuggingGPT 将用户指令分解为多模型协作的子任务

🔧 工具使用 (Tool Use)

工具使用是扩展 LLM 能力边界的关键手段。论文将工具使用分为三类：

• 工具选择：从工具集中选择最合适的工具（如 Toolformer, Gorilla）

• 工具创建：智能体自主创建新工具以解决新问题（如 CRAFT, ToolMaker）

• 工具推理：理解工具之间的依赖关系和组合方式（如 ToolChain*, ART）

核心挑战：巨大的动作空间中高效导航、工具组合爆炸、错误工具调用的恢复

🔍 搜索 (Search)

搜索是在推理空间中系统性地探索以找到最优解的方法：

• DFS/BFS：在思维树上进行深度/广度优先搜索

• MCTS：蒙特卡洛树搜索，通过模拟评估节点价值，平衡探索与利用

• Beam Search：在每步保留 top-k 候选，在效率和效果间取得平衡

• A* Search：启发式搜索，通过代价函数引导搜索方向（如 ToolChain*）

▲ 基础层智能体推理流程：目标→规划→工具使用→搜索验证→输出，含反馈循环

2.2 进化层：Self-Evolving Agentic Reasoning

进化层关注智能体如何通过反馈、记忆和自省来持续改进自身能力。与基础层的"稳定环境"假设不同，进化层承认环境是动态变化的，智能体必须从经验中学习并适应新情况。

📢 反馈学习 (Feedback Learning)

智能体从外部信号中学习改进策略：

• 环境反馈：执行动作后观察环境变化（如代码执行结果、网页响应）

• 奖励信号：来自奖励模型或人类评估的标量/偏好信号

• 自我评估：LLM 自身对输出质量的评判（如 Self-Refine 中的批评步骤）

关键挑战：稀疏奖励（长时间无反馈）、延迟奖励（最终结果才知好坏）、虚假反馈（误导性信号）

🧠 记忆系统 (Memory Systems)

记忆是智能体积累和复用经验的基础：

• 工作记忆（Working Memory）：当前对话上下文，短期、易丢失

• 情景记忆（Episodic Memory）：历史交互案例，可检索复用（如 Reflexion 中的经验库）

• 语义记忆（Semantic Memory）：抽象化的知识和规则，跨场景泛化

典型机制：MemoryBank 的记忆检索、Generative Agents 的反思链、Voyager 的技能库

🔄 自省与适应 (Self-Reflection & Adaptation)

智能体通过审视自身行为来识别错误和改进方向：

• Reflexion：将失败经验转化为语言反馈，存入记忆供未来参考

• Self-Refine：生成→批评→修改的迭代优化循环

• CRITIC：利用外部工具验证输出并修正错误

• Adaptation：根据环境变化调整策略，如 OPRO 的自动提示优化

2.3 集体层：Collective Multi-Agent Reasoning

集体层研究多个智能体如何通过协作、竞争或博弈实现超越单智能体的集体智能。本综述将多智能体推理分为三类架构： 集中式（Centralized，一个主控智能体分配任务）、去中心化（Decentralized，智能体平等协商）和层级式（Hierarchical，多层分工结构）。

🏗️ 集中式架构

一个中央控制器分配任务和协调执行。代表：MetaGPT（SOP驱动的角色分配）、AutoGen（可定制的智能体对话）。优点：协调高效；缺点：单点瓶颈。

🌐 去中心化架构

智能体之间平等交互，通过辩论、投票等机制达成共识。代表：CAMEL（角色扮演对话）、Multi-Agent Debate（多轮辩论）。优点：鲁棒性强；缺点：协调成本高。

🏢 层级式架构

高层智能体负责规划和决策，低层智能体负责执行。代表：ChatDev（CEO→CTO→程序员）、MADT（管理层→执行层）。优点：清晰分工；缺点：层级间信息丢失。

3 核心创新点深度解析

作为一篇综述论文，本论文的创新不在于提出新方法，而在于提出了全新的分类框架和洞见。以下是三个最核心的创新贡献：

创新1：三维分类框架 — 环境动态性 × 优化设置 × 智能体交互

此前的智能体推理综述通常只从单一维度（如只看规划方法、或只看多智能体系统）来组织文献，导致不同子领域之间的联系被忽视。本综述的三维分类使得研究者可以精确定位任何方法在全局中的位置，并发现跨维度的研究空白。例如，"Self-Evolving + Post-Training"这个交叉区域（通过 RL 训练智能体学习从反馈中进化）就存在大量研究机会。

创新2：In-Context Reasoning 与 Post-Training Reasoning 的系统区分

这个区分看似简单，实则深刻。它揭示了智能体推理优化的两条根本不同的路径：In-Context 路径通过更好的提示词、工作流编排和工具选择来"激发"LLM 已有的能力，无需额外训练但受限于模型固有能力上限；Post-Training 路径通过 RL/SFT/DPO 来"塑造"模型行为，可以突破能力上限但需要大量训练资源且容易过拟合。论文指出，最有效的系统通常结合两条路径——先通过训练建立基础智能体行为，再通过上下文编排优化具体任务的执行策略。

创新3：从推理到行动的统一路线图（Thought→Action 桥接）

论文最根本的贡献是提出了一个从"纯推理"到"推理-行动融合"的连续谱系。在这个谱系上：纯推理（CoT）→ 带工具的推理（ReAct）→ 自我改进的推理（Reflexion）→ 协作式推理（Multi-Agent Debate）逐步进化。每一步都不是对前一步的否定，而是在其基础上的能力扩展。这个视角帮助研究者理解：改进智能体推理不是寻找一个"终极方法"，而是在不同场景下选择合适的"进化阶段"。

5 关键技术细节

MCTS 在智能体推理中的应用

蒙特卡洛树搜索（MCTS）是智能体推理中最重要的搜索算法之一。其核心流程包含四个步骤：选择（Selection，用 UCB 公式选择最有价值的节点）→ 扩展（Expansion，生成新的子节点）→ 模拟（Simulation，从新节点随机模拟到终态）→ 回传（Backpropagation，将模拟结果沿路径更新节点价值）。

在 LLM 智能体推理中，MCTS 的每个节点代表一个"推理状态"（当前的思考链+环境观察），每条边代表一个"行动"（生成的推理步骤或工具调用）。UCB 公式平衡了对高价值节点的利用和对未充分探索节点的探索：

UCB(s,a) = Q(s,a) + c · √(ln N(s) / N(s,a))

其中 Q(s,a) 是状态 s 下行动 a 的平均价值，N(s) 是状态 s 的访问次数，N(s,a) 是行动 a 被选择的次数，c 是探索常数。这个公式确保了智能体不会一直沿着当前最优路径走（利用），也会尝试可能更优的替代路径（探索）。

多智能体协作的通信机制

多智能体系统的核心挑战之一是通信效率。论文总结了三种主要通信范式：

1. 共享记忆池：所有智能体读写同一块记忆区域（如 ChatDev 的共享文档），优点是信息完整，缺点是噪声大、扩展性差。

2. 消息传递：智能体之间点对点或广播消息（如 AutoGen 的对话机制），优点是灵活可扩展，缺点是可能遗漏关键信息。

3. 黑板系统：共享的"黑板"供智能体读写结构化信息，结合了前两者的优点——信息结构化且可扩展，但需要精心设计黑板的数据模式。

强化学习在智能体训练中的应用

论文详细讨论了 RL 在智能体推理中的应用，从两个角度展开：

从奖励信号来源：环境奖励（任务完成度）vs. 人类反馈（RLHF）vs. AI 反馈（RLAIF/Constitutional AI）。

从训练目标：结果奖励模型（ORM，只评估最终结果）vs. 过程奖励模型（PRM，评估每一步的正确性）。PRM 对于智能体推理尤其重要，因为长链条推理中只有结果奖励会导致信用分配困难。

7 设计精妙之处与局限性

✨ 设计精妙之处

1. 三维正交分类确保完备性

三个维度相互正交但不独立——它们从不同角度刻画同一事物，使得分类既无遗漏又不冗余。任何方法都能在三维空间中找到唯一位置。

2. In-Context vs. Post-Training 的实用指导

这个区分不只是学术分类，更是实践指南：无训练资源时选 In-Context 路径，需要突破能力天花板时走 Post-Training 路径，两者结合效果最好。

3. "层次递进"的环境动态性假设

基础→进化→集体的三层结构，隐含了设计哲学：高级能力建立在低级能力之上，不能跳过基础直接进化。

⚠️ 局限性

1. 综述时效性问题

智能体推理领域发展极快，2026年初至今已有大量新工作出现，这些内容无法被一篇2026年1月的综述覆盖。

2. 缺乏定量比较

对方法的比较主要是定性分类和描述，缺乏在统一基准上的定量实验对比。

3. 实际部署的鸿沟

许多方法在学术基准上有效，但在实际生产环境中面临延迟、成本、安全合规等巨大挑战。

8 Q&A 深度解析

Q1: Agentic Reasoning 和传统 LLM Reasoning 的本质区别是什么？
传统 LLM 推理（如 CoT、ToT）是"纯思维"活动——模型在内部生成推理链，不与外部环境交互。而 Agentic Reasoning 的本质区别在于三个要素：环境感知（能观察环境状态变化）、行动执行（能调用工具、执行动作）和迭代改进（能根据反馈修正策略）。这好比从"纸上谈兵"到"实战指挥"的转变——不是只想，而是边想边做边学。
Q2: In-Context Reasoning 和 Post-Training Reasoning 应该如何选择？
选择取决于两个因素：资源约束和性能需求。如果只有推理资源（无训练GPU），只能选 In-Context 路径——通过更好的提示词和工作流编排来优化。如果需要突破模型固有能力上限（如让模型学会新的工具使用策略），则必须走 Post-Training 路径。实践中最优方案是结合两者：先用 RL/SFT 建立基础智能体行为，再用 In-Context 编排适配具体任务。类比：Post-Training 相当于"练内功"，In-Context 相当于"用招式"——内功扎实才能使出精妙招式。
Q3: 为什么规划（Planning）是基础层的核心能力？
规划是连接"目标"和"行动"的桥梁。没有规划，智能体只能做出局部最优的反应式决策，无法完成需要多步协调的复杂任务。规划让智能体能够"向前看"——预判未来几步的后果，从而做出全局更优的决策。更重要的是，规划能力是进化层（自我改进）和集体层（多智能体协调）的前提——你必须先能规划自己的行动，才能改进规划或与他人协调规划。
Q4: 情景记忆和语义记忆在智能体推理中分别扮演什么角色？
情景记忆存储具体的交互案例（如"上次执行某API调用时返回了什么错误"），它是具体的、实例级的，检索相似案例来指导当前决策——类似人类从"上次经历"中学习。语义记忆存储抽象化的知识和规则（如"当遇到A类问题时，应使用B策略"），它是泛化的、跨场景的，允许智能体将经验抽象为可迁移的知识——类似人类从"多次经历"中总结"规律"。两者互补：情景记忆提供具体参考，语义记忆提供通用指导。
Q5: 多智能体系统一定优于单智能体吗？
不一定。多智能体系统的优势在于专业分工（每个智能体专注一个领域）和冗余验证（多个智能体交叉检查减少错误）。但它也引入了额外成本：通信开销（智能体间信息传递需要 token）、协调难度（需要共识机制避免冲突）、错误传播（一个智能体的错误可能影响其他智能体）。对于简单任务，单智能体 + 好的提示词往往更高效。多智能体的价值在任务复杂度超过单智能体处理能力时才真正显现。
Q6: MCTS 在 LLM 推理中的优势和局限分别是什么？
优势：MCTS 平衡了探索与利用，可以在巨大的推理空间中高效搜索；通过模拟评估节点价值，不需要显式的价值函数；天然支持多分支探索，不会陷入局部最优。局限：MCTS 的搜索效率严重依赖模拟质量——如果 LLM 的模拟（rollout）质量差，搜索就会被误导；此外 MCTS 的计算开销大（需要多次 LLM 调用），在严格延迟约束下不适用；最后，MCTS 需要精心设计奖励函数，而许多任务的奖励信号稀疏且难以定义。
Q7: 过程奖励模型（PRM）为什么对智能体推理特别重要？
智能体推理通常是长链条的——从目标分解到工具调用到结果验证可能包含数十个步骤。如果只有结果奖励（ORM），模型只知道最终对不对，无法判断哪一步出了问题。这就像考试只看总分，不看每道题的对错——你不知道该复习哪部分。PRM 为每一步提供细粒度反馈，使得模型能够精确定位错误步骤并针对性改进，大大提高了学习效率和最终性能。
Q8: 本综述指出的最重要的未来方向是什么？
论文列出了五个关键未来方向：(1) 个性化——让智能体适应用户偏好和习惯；(2) 长视野交互——支持智能体在数天甚至数周的持续任务中保持连贯性；(3) 世界建模——让智能体构建内部世界模型以预测行动后果，而非纯试错；(4) 可扩展的多智能体训练——如何高效训练大量智能体协同工作；(5) 现实世界部署的治理——安全、伦理、可控性等社会技术问题。其中，世界建模可能是最根本的——拥有准确世界模型的智能体可以在"脑海中"模拟行动后果，大幅减少实际试错成本。
