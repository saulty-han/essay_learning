1 研究动机：种群进化易陷局部最优

如何为组合优化等复杂任务自动设计高质量启发式函数（heuristic）而无需大量领域专家知识？现有 LLM-based AHD（如 EoH、Funsearch、ReEvo）采用基于种群（population-based）的进化计算，维护固定数量的精英函数，会直接丢弃低性能函数——但低性能函数经多步改良后可能变优，使进化陷入局部最优。

种群结构的缺陷

种群方法维护精英集，红虚线之下的低性能函数被直接丢弃。这种"赢者通吃"机制使启发式空间探索不充分，难以跳出局部最优。

本文主张

首次将 MCTS 应用于 LLM-based AHD：用树结构保存所有已生成启发式，能"暂时在劣势函数上继续开发"，实现更全面的启发式空间探索。

核心创新

把 LLM 生成的所有启发式组织成一棵 MCTS 树（根节点为虚拟节点），通过 Selection / Expansion / Simulation / Backpropagation 全面探索，并引入 tree-path reasoning 等树特有动作。

2 框架概览：启发式 MCTS 树

MCTS-AHD 将 LLM 生成的每个启发式函数组织成 MCTS 树；根节点 nr 为虚拟节点，其余每个节点代表一个可执行的 Python 启发式函数 h∈H 及其语言描述。反复执行四阶段直到评估次数达上限 T。

端到端四阶段

初始化 NI 个初始节点 → Selection（归一化 UCT 选到叶）→ Expansion（LLM 动作生成 2k+2 子节点）→ Simulation（评估 g(h) 更新 Q/N/精英集）→ Backpropagation（Q←max 子节点）。终止时输出全树中性能最高的启发式。

3 MCTS 探索空间：LLM 动作集

LLM-based Actions 模拟 EC 的变异 / 交叉算子，并提供一种 MCTS 特有的树路径推理动作。单次扩展生成 2k+2 个子节点：m1、m2 各 k 个，e2、s1 各 1 个。

动作集合

i1 Initialization（从零生成）；m1/m2 Mutation（m1 引入新机制 / 公式，m2 修改参数）；e1 Crossover（从多个函数生成全新工作流）；e2 Crossover（以 parent 为基，参考精英集 E 前 10 的函数改进）；s1 Tree-path Reasoning：分析叶到根所有唯一启发式，识别优势设计并生成增强函数——这是 MCTS 特有、依赖树结构的动作。

4 启发式保留与演化

Thought-Alignment Process 解决 LLM 幻觉导致的"代码与描述不一致"问题。每个动作调用 LLM 两次：第一次生成 Python 实现，第二次基于已生成代码用最多 3 句话总结描述。

Thought-Alignment

第二次调用显著更短，不会显著增加时间 / Token 成本，却使语言描述与代码高度相关，提升后续 tree-path reasoning 的输入质量。

节点质量值

节点质量 Q 等于其启发式在评估集 D 上的性能 g(h)；Selection 用归一化 UCT：UCT(c) = (Q(c)−qmin)/(qmax−qmin) + λ·√(ln(N(nc)+1)/N(c))。

5 低性能启发式的复用

Exploration-Decay（探索衰减）

前期鼓励探索、后期保证收敛。公式 λ = λ0·(T−t)/T（式 7），λ0 默认 0.1（对所有任务通用，避免任务特定调参），α=0.5。这使得被低估的低性能函数能在前期得到充分开发机会。

Progressive Widening

当 ⌊N(n)^α⌋ ≥ |children(nc)|（α=0.5）时新增子节点；根节点满足条件时调用 e1（从 2–5 个不同子树均匀选函数），其他节点调用 e2。这让树随评估预算渐进扩展，保留并发展暂时低性能的启发式。

6 树搜索与进化的整合

端到端数据流

Selection → Expansion（e2/m1/m2/s1 生成 2k+2 子节点）→ Simulation（评估 g(h)，更新 Q/N/精英集 E/qmax/qmin）→ Backpropagation（Q(nc)←max_{c∈Children}Q(c)，N(nc)←ΣN(c)）。性能评估次数接近 T（默认 1000）时停止，输出全树最佳启发式。

7 实验评估：TSP / KP / BPP 等

设置与基线

任务：TSP、KP、CVRP、MKP、BPP、ASP 等 NP-hard 组合优化；评价预算 T=1000；每个方法独立运行 3 次取平均。基线：手工启发式（Nearest-greedy/ACO/EI）、GHPP、POMO/DeepACO（NCO）、Funsearch/EoH/ReEvo/HSEvo（LLM-AHD）。LLM 用 GPT-3.5-turbo 与 GPT-4o-mini。

主结果：TSP N=50 Gap：MCTS-AHD 9.69% vs EoH 12.67% vs Funsearch 12.00%；KP N=100/W=25 Gap：MCTS-AHD 0.04% 最优；online BPP 平均 Gap 0.89% vs EoH 1.17% vs ReEvo 1.71% vs HSEvo 1.13%。TSP/KP 用 T=1000 约 3 小时、1M 输入 Token、约 0.3 美元。

8 敏感性分析：消融

组件消融（TSP50/KP100）

原始 MCTS-AHD：TSP 10.661% / KP 0.059%；w/o Progressive Widening：12.132% / 0.064%；w/o Thought-alignment：11.640% / 0.061%；w/o Exploration-decay：11.606%。三个组件均有正向贡献，Progressive Widening 影响最大。

10 总结与未来展望

MCTS-AHD 首次把 MCTS 用于 LLM 自动启发式设计，用树结构保留并发展所有启发式（含低性能者），通过 tree-path reasoning、Thought-Alignment、Exploration-Decay 与 Progressive Widening 实现全面探索，在 TSP/KP/BPP 等任务上全面超越 EoH/ReEvo/Funsearch。

Q1：为什么 MCTS 比种群进化好？
种群直接丢弃低性能函数易陷局部最优；MCTS 用树保存所有启发式并渐进扩展，可继续开发被低估的劣势函数。
Q2：tree-path reasoning 是什么？
分析叶到根所有唯一启发式，识别优势设计生成增强函数，是依赖树结构的 MCTS 特有动作。
Q3：Exploration-Decay 如何工作？
λ=λ0·(T−t)/T，前期大 λ 鼓励探索、后期小 λ 保证收敛，λ0=0.1 通用无需调参。

局限与未来

依赖 LLM 调用成本与质量；树规模随预算增长；目前验证于组合优化任务。未来可探索更高效的扩展策略与跨任务迁移。
