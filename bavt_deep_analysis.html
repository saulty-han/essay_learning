Spend Less, Reason Better

预算感知价值树搜索（BAVT）— 让 LLM 智能体在有限预算下精打细算地推理

📄 arXiv Preprint 2026  |  arXiv: 2603.12634  |  GitHub: bavt-budget-aware-agent

Test-Time Scaling Budget-Aware Value Tree Search Multi-Hop QA Training-Free

★ 整体架构总览

BAVT（Budget-Aware Value Tree）是一个免训练的推理时框架，核心思想是将多跳推理建模为一棵动态搜索树，由步骤级价值估计引导搜索方向，并通过预算条件节点选择和残差价值预测来在有限计算预算下最大化推理质量。

▲ BAVT 架构总览：动态搜索树 + 步骤级价值估计 + 预算条件选择 + 理论收敛保证

📊 残差价值预测

评估每一步的"相对进度"而非"绝对质量"，克服 LLM 自评过度自信的问题。只关注"这一步让我离目标近了多少"，而非"当前状态有多好"。

💰 预算条件选择

用剩余资源比率作为价值缩放指数，实现从广泛探索到贪婪利用的无参数自然过渡。预算充裕时大胆探索，预算紧张时聚焦最优。

📐 理论收敛保证

在显式有限预算下证明 BAVT 以至少 1-ε 的概率收敛到最终答案，是少数具有理论保证的推理时缩放方法。

0 核心问题与研究动机

测试时缩放（Test-Time Scaling）已成为提升 LLM 智能体可靠性的主流范式——给模型更多推理时间，让它在多个候选答案中选择最好的。然而，现有方法有一个共同的盲点：把计算当成无限资源。无论是重复采样（best-of-n）还是搜索树（MCTS），都在"用尽预算"或"达到步数上限"时粗暴停止，完全不考虑中间步骤的性价比。

🔑 核心问题

如何在严格的计算预算约束下，让 LLM 智能体高效地完成多跳推理？即：每花一个 token/一次工具调用，都要确保它"物有所值"——避免在死胡同轨迹上浪费资源，将预算集中在最有希望的推理路径上。

BAVT 的核心洞察是：智能预算管理 > 暴力计算扩展。与其给4倍资源让基线方法浪费，不如在1倍资源下精打细算——通过步骤级的价值评估实时判断哪条路径值得继续投入，哪条应该立即剪枝，让每一分预算都花在刀刃上。

1 背景知识与前置概念

测试时缩放 (Test-Time Scaling)

测试时缩放指在推理阶段分配更多计算资源来提升模型表现。主要有三类方法：(1) 后训练微调——通过 RL/SFT 训练模型学习更好的推理策略（如 o1、DeepSeek-R1）；(2) 奖励引导的 CoT 生成——在推理过程中用奖励模型引导搜索方向（如 MCTS + PRM）；(3) 多样本生成——重复采样多个候选答案再选择最优（如 best-of-n、self-consistency）。BAVT 属于第(2)类和第(3)类的融合——通过搜索树探索多种路径，同时用价值估计引导搜索方向。

LLM 自评的过度自信问题

研究表明，LLM 在评估自身输出质量时存在系统性过度自信——即使是错误的推理步骤，LLM 也倾向于给出高分。这导致基于 LLM 自评的价值估计不可靠。BAVT 通过残差价值预测来缓解这个问题：不评估"当前状态有多好"（容易过度自信），而是评估"这一步相比上一步进步了多少"——一个相对量，对过度自信更鲁棒。

多跳推理 (Multi-Hop Reasoning)

多跳推理指需要多次信息检索和推理步骤才能回答的问题。例如"获得2024年诺贝尔物理学奖的科学家毕业于哪所大学？"需要两跳：(1) 查出获奖者是谁；(2) 查出该科学家的毕业院校。每一步都可能出错，错误会传播到后续步骤，因此需要在每一步评估进度并及时纠偏——这正是 BAVT 的核心用武之地。

2 方法论：BAVT 三大核心模块

2.1 动态搜索树构建

BAVT 将多跳推理建模为一棵动态搜索树：根节点是原始问题，每个子节点代表一个推理步骤（含工具调用和观察结果）。树的构建是"按需"的——每次选择当前最有价值的叶节点进行扩展，生成新的推理步骤作为子节点。关键设计：树在单一 LLM 主干内构建，不需要额外的价值模型或奖励模型——价值估计由同一 LLM 完成。

2.2 残差价值预测器 (Residual Value Predictor, RVP)

RVP 是 BAVT 解决 LLM 过度自信的关键。传统方法让 LLM 评估"当前推理链的质量"（绝对价值），LLM 倾向于给出高分。RVP 改为让 LLM 评估"最近一步推理让我离答案近了多少"（残差价值）：

ΔV(t) = V(st) - V(st-1)

这个残差设计的好处是：即使 LLM 对绝对价值 V(st) 估计偏高，只要偏差在不同步骤间相对稳定，差分后的 ΔV 仍然能反映真实的进步程度。这就像用"温度变化"而非"当前温度"来判断加热是否有效——即使温度计有系统偏差，变化量仍然可信。

2.3 预算条件节点选择 (Budget-Conditioned Node Selection)

选择下一个扩展节点时，BAVT 不仅考虑节点的价值，还考虑剩余预算。评分公式：

Score(node) = V(node) × (Bremaining / Btotal)α

其中 Bremaining / Btotal 是剩余预算比率。当预算充裕（比率接近1），分数≈V(node)，价值高的节点被优先选择（探索）；当预算紧张（比率趋近0），分数被大幅压缩，不同节点间的价值差距缩小，搜索变得更"谨慎"——避免在低价值节点上浪费最后一点预算。α 控制过渡的速度，论文发现 α=1（即直接用剩余预算比率）效果最好且无需调参。

3 核心创新点深度解析

创新1：残差价值预测 — 从绝对评估到相对进度

这是 BAVT 最核心的技术创新。LLM 自评过度自信的根本原因是：LLM 很难区分"看起来合理的推理"和"真正正确的推理"——两者在语言流畅度上几乎没有区别。但 LLM 对相对变化更敏感——"这一步是否让我获得了新信息"比"当前推理链是否完全正确"更容易判断。残差价值预测利用了这个不对称性。

创新2：预算条件选择 — 无参数的探索-利用过渡

传统搜索算法中的探索-利用平衡通常需要手动调节温度参数或探索常数。BAVT 的预算条件选择巧妙地将剩余预算比率作为自然缩放指数——不需要任何额外超参数。这个设计的美妙之处在于：随着搜索进行，剩余预算自然递减，搜索策略自然从"广泛探索"过渡到"聚焦利用"，完全不需要人为干预。

创新3：免训练 + 理论收敛保证

大多数推理时缩放方法要么需要训练价值模型（如 PRM），要么缺乏理论保证（如 MCTS 的收敛性依赖模拟质量）。BAVT 是少数兼具免训练和理论收敛保证的方法：它只需要 LLM 自身做价值估计（无需额外模型），并且在有限预算下以高概率收敛到最终答案。这个保证基于搜索树价值的单调递增性——每次扩展都选择当前最优节点，因此最终答案的质量不会比中间任何节点差。

5 关键技术细节与理论分析

收敛性定理

BAVT 的理论保证可以形式化为：给定总预算 B 和容差 ε > 0，BAVT 在 B 步内以至少 1-ε 的概率找到最终答案。证明的关键步骤：

(1) 搜索树的价值 V(最优叶节点) 随扩展次数单调递增（因为每次选择当前最优节点扩展）

(2) 预算条件选择确保在有限步内必定终止（因为剩余预算单调递减至0）

(3) 残差价值估计的误差有界（假设 LLM 价值估计的偏差不超过常数 C），因此累计误差可控

这个保证虽然在实际中可能因 LLM 价值估计的不完美而打折扣，但提供了理论上的可靠性锚点。

剪枝策略：低效工具调用的自动识别

BAVT 通过残差价值自动识别和剪枝低效的推理路径。当某个节点的 ΔV 接近0或为负时，意味着"这一步几乎没有让我离答案更近"——这样的路径会被低优先级排序，自然被"饿死"（不被选中扩展）。这比显式剪枝更优雅——不需要设定阈值，预算压力自然会淘汰低效路径。

7 设计精妙之处与局限性

✨ 设计精妙之处

1. 残差价值的抗过度自信设计

通过差分消除 LLM 自评的系统偏差，巧妙利用了偏差的相对稳定性——绝对值不准，但变化量可信。

2. 无参数的探索-利用过渡

剩余预算比率是天然的缩放因子，随搜索进行自然从探索过渡到利用，无需手动调参。

3. 理论与实用的平衡

既有理论收敛保证，又不需要训练额外模型，实用性很强。

⚠️ 局限性

1. 仅在 QA 任务上验证

实验限于多跳 QA 基准，未在代码生成、具身推理等更复杂场景验证。

2. LLM 价值估计仍不完美

残差价值缓解了过度自信，但 LLM 的价值判断仍可能出错，尤其是步骤间差异很小时。

3. 搜索开销

每步需要多次 LLM 调用（生成+评估+选择），延迟较高。

8 Q&A 深度解析

Q1: BAVT 和标准 MCTS 的核心区别是什么？
标准 MCTS 用固定的 UCB 公式选择节点，探索常数需要手动设定，且不考虑预算约束。BAVT 做了两个关键改造：(1) 用残差价值替代绝对价值，更可靠地估计节点质量；(2) 用预算条件选择替代 UCB，根据剩余资源自动调整探索强度。此外，MCTS 通常需要外部奖励模型，而 BAVT 由 LLM 自身完成价值估计，完全免训练。
Q2: 残差价值为什么比绝对价值更鲁棒？
假设 LLM 对每一步的绝对价值评估都偏高一个常数 C（过度自信），那么 ΔV = (V_real + C) - (V_prev + C) = V_real - V_prev。常数偏差在差分中被消除了！当然实际偏差不是完全恒定的，但研究表明 LLM 的过度自信偏差在相似步骤间相对稳定，因此差分能显著降低偏差影响。
Q3: 预算如何定义？是 token 数还是工具调用次数？
论文中使用的是综合预算定义：B = α × token消耗 + β × 工具调用次数。两个分量的权重可以根据实际场景调整。对于 API 成本敏感的场景，可以加大工具调用次数的权重；对于延迟敏感的场景，可以加大 token 消耗的权重。灵活性是 BAVT 设计的一个优点。
Q4: BAVT 能否与 RL 训练结合？
可以。虽然 BAVT 本身是免训练的，但可以通过 RL 训练 LLM 生成更高质量的推理步骤和更准确的价值估计。训练后的 LLM 作为 BAVT 的主干，价值估计更准确，搜索效率更高。这是一个自然的增强方向。
Q5: "智能预算管理 > 暴力扩展"的结论有多通用？
这个结论在多跳 QA 任务上得到了验证，但其通用性需要更多实验确认。直觉上，对于"步骤质量差异大"的任务（有的步骤价值很高，有的几乎无用），智能预算管理的优势最大——因为剪枝低效步骤的收益大。对于"每步价值差不多"的任务，智能管理和暴力扩展的差异可能较小。
Q6: BAVT 在低预算下的优势从何而来？
低预算时（如1×），暴力扩展方法（Best-of-N、标准MCTS）的资源不够覆盖足够的搜索空间，往往在到达有希望的路径之前就耗尽了预算。BAVT 的预算条件选择确保在低预算时聚焦最有希望的路径——从一开始就"精打细算"，不浪费任何一步。这就是为什么 BAVT 在1×预算下能超过4×预算的基线。
Q7: BAVT 的理论保证在实际中有多可靠？
理论保证假设 LLM 的价值估计偏差有界，这在实际中通常成立但不完美。如果 LLM 对某步的价值严重高估，可能导致搜索被误导。不过，残差价值设计已经大幅缓解了这个问题。论文的经验结果显示，BAVT 在绝大多数情况下确实能找到高质量答案，理论保证与实践表现基本一致。
Q8: BAVT 与 DeepConf 的早停机制有什么区别？
DeepConf 的早停基于模型自身的 logprobs 置信度——如果当前生成的 token 置信度低就停止。BAVT 的"早停"是基于搜索树的价值评估和预算条件——如果某条路径的残差价值低且预算紧张，就不在这条路径上继续投入。前者是生成级别的（在 token 层面），后者是推理步骤级别的（在搜索路径层面），粒度和机制完全不同。

9 代码使用与实战

# 克隆 BAVT 仓库
git clone https://github.com/mlbrilliance/bavt-budget-aware-agent.git
cd bavt-budget-aware-agent

# 安装依赖
pip install -r requirements.txt

# 仓库结构
bavt-budget-aware-agent/
├── bavt/
│   ├── search_tree.py      # 动态搜索树
│   ├── value_estimator.py  # 残差价值预测器
│   ├── budget_selector.py  # 预算条件节点选择
│   └── agent.py           # BAVT 智能体主类
├── eval/               # 评估脚本
├── data/               # 基准数据集
└── configs/            # 配置文件
# 运行 BAVT 在 HotpotQA 上
python run.py \
  --dataset hotpotqa \
  --model meta-llama/Llama-3-70B \
  --method bavt \
  --budget 1.0 \
  --residual_value True

# 对比基线
python run.py --dataset hotpotqa --model meta-llama/Llama-3-70B --method best_of_n --budget 4.0
python run.py --dataset hotpotqa --model meta-llama/Llama-3-70B --method mcts --budget 4.0
参数默认值说明
--budget1.0预算倍率（1.0=CoT 等价）
--residual_valueTrue是否使用残差价值预测
--alpha1.0预算缩放指数
--max_depth8搜索树最大深度
--beam_width3每步扩展的候选数
